## Cross-cutting wiring

- `security-guard` ↔ `build-doctor` — release flow is gated. The `agent-zero-build`
  skill must invoke `release-build-pipeline` before tagging. Pinned in project memory
  (`memory/project_release_security_gate.md`).
- `code-coach` ↔ `git commit` — auto-invoke condition documented in
  `engine/pre-commit-review.md` and pinned in project memory
  (`memory/project_pre_commit_code_coach.md`).
- All four specialists rely on `knowledge/agentzerolite-architecture.md` as the shared
  ground truth so file:line findings stay consistent.

## Trigger registration

All triggers are **project-specific** (refer to .NET / Akka / WPF / ConPTY /
AgentChatBot / release pipeline) — registered exclusively via frontmatter
`triggers:` on each agent/engine MD. **No changes to the standard
`harness-kakashi-creator` SKILL.md.** This follows the
`references/skill-separation.md` rule: standard skill stays general; project
specifics stay in `harness/`.
