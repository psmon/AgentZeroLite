  - Use **VS 2022's bundled CMake**, not MSYS2's
  - Use the `"Visual Studio 17 2022" -A x64` generator → MSBuild backend
  - `cmake --build … -j` parallelism is handled by **MSBuild natively on
    Windows**, never by `make -jN` under MSYS2 fork emulation

  Bash stays usable as a thin command wrapper, but never runs the heavy
  compilation. The MSYS2 fork path is no longer exercised at scale, which
  removes the entire class of crashes the original stackdump represented.
