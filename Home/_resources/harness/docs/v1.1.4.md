# v1.1.4 — code-coach GitHub issue handoff policy

**Date**: 2026-04-27
**Bump**: patch (workflow refinement, no engine/agent count change)

## What changed

### Policy

When `code-coach` (Mode 1 manual or Mode 2 pre-commit) produces **any
finding at Suggestion severity or higher**, the agent now files a GitHub
issue tracking those findings before the review is considered complete.
Pure-pass reviews (zero findings) skip this step.

The commit itself is not gated on issue creation — the issue exists to
make sure advisory items survive after the merge, not to block the merge.

## Why

Before v1.1.4, advisory suggestions disappeared once a commit went in.
The Coach said "S-5 would be nice" → user said "ignore and commit" → the
suggestion lived only in the log file and was never acted on.

Tracking issues solve two problems:

1. **Visibility** — `gh issue list` shows the cleanup backlog at any time.
2. **Accountability** — the issue has a "Closes-when" checklist, so each
   suggestion has a defined exit condition rather than rotting in a log.

The threshold deliberately starts at Suggestion (the lowest severity) so
the policy is *opt-out* rather than *opt-in*: every finding gets a track,
and only zero-finding reviews stay log-only.

## Migration impact

None. Workflow-only change — no code, build, or DB impact. Existing
reviews under `harness/logs/code-coach/` are unaffected; only future
reviews follow the new procedure.

## Reverse rollout

If issue noise becomes a problem (e.g. too many low-value Suggestion-only
issues), revert by:

1. Drop step 6 from Mode 1 + Mode 2 in `code-coach.md`
2. Drop the "GitHub issue handoff" section
3. Drop the issue bullet from `pre-commit-review.md` Output
4. Drop step 5 from `memory/project_pre_commit_code_coach.md`
5. Bump to v1.1.5 with rollback notes
6. Existing issues stay as-is — close them manually or let them age out

Alternative graduation: raise the threshold from Suggestion to Should-fix
(only file issues for the genuinely actionable findings).
