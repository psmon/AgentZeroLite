M0011 — Token Remaining (Widget)

Minimal claude-hud-style widget. Architecture A: AgentZero installs a tiny statusLine wrapper that tees Claude Code's stdin (rate_limits only) into per-account snapshot files; widget polls every 60 s. claude-hud is detected at install time and piped through (so users keep their statusline). Per-account install/uninstall with full backup/restore of ~/.claude/settings.json.

Scope: this widget shows Usage (5h) and Weekly (7d) — both per-account aggregates. Context % is intentionally excluded — it's per-session and belongs in a different surface.

5) Install / Uninstall dialog

Token Remaining — statusLine wrapper installer

Per-account install. The wrapper captures Claude Code's rate-limit telemetry (Usage 5h, Weekly 7d) so the widget can read it. Your existing statusLine (claude-hud, etc.) is preserved and piped through automatically.

What gets installed (per account)

1. Backup current settings.json → %LOCALAPPDATA%\AgentZeroLite\statusline-backup\\settings.json..bak 2. Replace settings.json statusLine.command with: "%LOCALAPPDATA%\AgentZeroLite\statusline\az-hud-wrapper.cmd" --account claude-pel3 3. Wrapper writes per-tick snapshot to: %LOCALAPPDATA%\AgentZeroLite\cc-hud-snapshots\claude-pel3.json 4. If claude-hud detected, wrapper appends "node " as pipe target so user's statusline is preserved.

Uninstall — restore preview

For account "claude": current statusLine: az-hud-wrapper.cmd --account claude --pipe claude-hud ↓ restore from .bak (sha256 verified) ↓ original statusLine: { type: "command", command: "node ${CLAUDE_PLUGIN_ROOT}/dist/index.js" } If backup hash doesn't match (user edited settings since install), operator is shown a 3-way diff and confirms before any write. Snapshot files in cc-hud-snapshots/ are kept (orphaned, but small).
Settings.json is parsed via System.Text.Json with comments allowed; written atomically via temp+rename. Wrapper script is read-only after install.
