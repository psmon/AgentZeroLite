# 요청 (Brief)

음악재생 목록에 유튜브 외에 추가로 로컬 MP3를 스캔해 분류하고,
MP3 음원을 재생하는 플레이리스트를 추가한다.

- **재생 모드**: MP3 재생은 vision이 없기 때문에 걸그룹 재생 모드가 아닌
  일반 재생 환경(악기·가수) 기반 플레이어 모드로만 작동 (일반 모드는 이미 구현됨).
- **재생목록 분리·영속**: 현재 유튜브 재생목록과 분리된 별도 재생 리스트를
  내부 SQLite로 관리 — 마이그레이션·재설치 시에도 재생목록 유지.
- **음악 분류**: 스캔 시 MP3 태그 + 파일명 + 폴더명을 근거로,
  이미 보유한 LLM을 이용해 분류한다.
- **UI**: 상단에서 재생목록 소스(유튜브 / MP3 플레이어)를 선택하는 탭 구성.
  어떤 소스를 선택하든 하단에는 기존 구축된 시각화 밴드가 동일하게 작동.

## 확장 요청 (운영자 추가, 2026-07-05)

- MP3 재생 시 환경음(악기)을 파악할 수 있으므로(AST 루프백 분류),
  **재생 중 트랙의 보유 악기를 리얼타임으로 업데이트**해 영속화한다.
- MP3 플레이어 리스트에서 **악기 필터명으로 검색** 지원 —
  예: "피아노 포함", "피아노만 있음(단독)" 같은 검색 필터.

## 확장 요청 2 (운영자 추가, 2026-07-05)

- 스캔은 오래 걸릴 수 있으므로 **비동기 백그라운드 잡(task job)** 형태로 진행.
- 스캔이 도는 중에도 플레이어 리스트가 **증분 업데이트** — 스캐닝된 데이터만큼
  즉시 재생 가능해야 함.
