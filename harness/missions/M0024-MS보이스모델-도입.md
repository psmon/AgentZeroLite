# 요청 (Brief)

기존 보이스 모델은 stt는 발언자 구분이 안되었습니다.
보이스모델에 다음을 분석 추가 모델로 영입해주세요
- https://github.com/microsoft/VibeVoice

권장:가급적 pip 설치없이 최근 music 모델을 도입했듯이 내장 모델 AgentZero Common에서 처리될수 있게해주세요
(유사하게)/

이 모델은 음성노트에 발언자를 구분하는기능을 도입하는게 우선입니다.
보이스 테스트 페이지에서 발언자 구분가능모델인경우 구분표시
-소스는 마이크입력 모드외에 pcm 직접 모드도 가능하게 해주세요
웹플로그인 보이스 노트에도 해당모델을 사용하는경우 발언자 구분가능하게 개선

## Acceptance

- [ ] VibeVoice 실제 capability 확인 — speaker diarization 지원 여부 명확화

- [ ] Voice 테스트 입력 소스에 **PCM 직접 모드** 추가 (Music 탭 WASAPI 루프백 패턴 재사용)

## Notes

- 우선순위: 음성노트 화자 구분 (operator 명시)
- "Music 모델 도입한 것처럼" — Music 탭의 `IMusicClassifier` + `OnnxAstClassifier` + `AstModelDownloader` + `MelSpectrogram` 구조를 reference 로 삼을 것. 같은 seam 모양으로 만들면 향후 pyannote / whisperX 등 다른 화자 구분 모델도 같은 슬롯으로 교체 가능.
- 단계별 진행 — 1단계 (research + scaffolding) 끝나면 operator confirm 받고 본격 통합 진행.
