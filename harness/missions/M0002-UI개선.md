# 요청 (원문)

스킬이용 : harness-kakashi-creator

아래와 같이 UI/UX를 개선하고자함

- 설정 셋팅창에서 닫기 x버튼이 너무 작아 인지하기 어려움 X보나.  CLOSE 버튼으로 변경하고 인지가능하게 색상 적용( 동일다크테마로 x버튼 찾기어려웠음 인지가능한 이질감없는 색상을 사용하되 동일하게 다크테마 이용하면 찾기 어려울수 있는점고려)
  -CLI 설정에서 ExePath고를시 선택(콤보)에서 비 다크테마여서 인지하기 어려움 ex>바탕은 흰색,글자는 그레이 색상 인지하기 어려워 뭐를 선택했는지 인지 불가.. 다크테마를 제대로 적용해 일관성+UI인지 향상 ( 추가- CLI EDIT대상을 선택하기 위해 선택 효과도 인지하기 어려움 ..완전하게 반전효과가 필요 )
  ( CLI 설정의 전반적인 일관성있는 다크테마 적용을 검토해볼것)

- LLM 모델 선택창도 위와 유사( 콤보-선택창이 전반적으로 인지하기어려운 이질적 테마적용됨)

-터미널을 추가하면 탭윈도우 좌측은 자신의 이름뜨며 우클릭시 팝업테마 는 OK
> 우측에 버튼이 하나있는데 누르면 탭전환이 있음 이 기능은 무의미함 탭전환기능은 이미 있어서, 이영역에 창분리 기능을 적용할것 누르면 도킹이 분리되어 새창모드가 됨 그리고 새창이 되고 나서 다시 돌아오는 버튼도 구현해 복원되도록

## Notes

- UI 변경은 다크테마 일관성을 깨뜨리지 않는 범위에서 한다.
- CLOSE 버튼 색상은 dark surface와 충돌하지 않으면서도 X 보다 인지가 쉬운 톤(예: 약간 밝은 회색 + 호버 시 강조).
- 분리/복원 기능은 ConPTY 세션의 수명을 끊지 않도록 주의 — TerminalActor와 ITerminalSession은 그대로 두고, 호스팅 컨테이너만 옮기는 방식으로 구현.
