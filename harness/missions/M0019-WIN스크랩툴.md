## Notes

- 본 미션은 **다중 follow-up fix** 가 자연스럽게 일어나는 구조다 (mission-dispatch
  §10 패턴). Stage 1 종료 → 운영자 smoke test → Stage 2 / 3 / 4 / 5 각각
  `## 후속 수정 #N` 으로 동일 완료 로그에 append.
- 클래스명을 `AgentBotActor` → `AgentCliActor` 식으로 바꾸지 않는 이유:
  Akka 액터 경로 (`/user/stage/bot`) 가 안정 contract 이고 모든 곳에서 참조됨.
  *user-facing label* 만 변경하는 것이 부작용 0, 의도 전달 100%.
- Scrap module 의 `os automation` 기능과의 중복 분석 — `os element-tree`,
  `os screenshot` 는 이미 존재. Scrap 의 *차별점* 은 "텍스트 캡처 + 스크롤
  자동화 + UIA TextPattern 풀세트". 따라서 Scrap 과 os 는 *공존* 한다 (os 가
  read-only 셸 도구, Scrap 이 풀 capture UI + 텍스트 캡처).
