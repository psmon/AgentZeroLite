## What's new

### `harness/knowledge/code-coach/nodejs-bash-and-stream-io-pitfalls.md`

10-pitfall catalogue capturing every failure mode found while building
the M0011 token-remaining statusLine wrapper. Each pitfall has:

- **Symptom** — the actual log line operators saw
- **Root cause** — what's really happening
- **Fix** — copy-paste code snippet
- **Pre-commit check** — what `code-coach` looks for in a diff

Pitfall index:

| # | Topic | One-line |
|---|---|---|
| 1 | `bash` PATH ambiguity | WSL bash returns `0xC0000142` from non-interactive parents; pin Git Bash explicitly |
| 2 | `cp.spawn(_, _, {shell:true})` | cmd /c shreds bash `'…'` nesting; parse argv ourselves |
| 3 | Git Bash needs MSYS HOME | `C:\Users\x` → `/c/Users/x` so `~/...` globs work |
| 4 | Claude Code `${VAR}` / `$(...)` substitution | Applied even inside single quotes — sidecar file pattern for complex payloads |
| 5 | Inheriting child stderr | Diagnosis-killer; capture to log |
| 6 | `child.stdin.write` early-close | EPIPE crashes wrapper; error-guard + try/catch |
| 7 | Non-atomic file writes | Reader sees half-written JSON; temp + rename + copy fallback |
| 8 | Unbounded diagnostic logs | Self-trim every write; tail-N keep |
| 9 | Windows native exit codes | Humanise NTSTATUS (`0xC0000142`, `0xC0000005`, …) at log time |
| 10 | Invasive installer rollback | sha256 of original + diff-then-confirm uninstall |

The doc closes with a paste-into-PR-description checklist (10 items)
and cross-references the M0011 mission record + WPF pitfalls catalogue
(the same pattern, different domain).
