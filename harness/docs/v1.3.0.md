# v1.3.0 — Missions subsystem (file-based request inbox)

**Released**: 2026-05-02
**Bump rationale**: Minor — new subsystem (`harness/missions/` +
`harness/logs/미션기록/`), tamer agent extended with mission-dispatch triggers,
new knowledge document. No breaking changes; existing logs and agents continue
to function unchanged.

## What's new — at a glance

```
harness/
├── missions/                                  ← NEW: operator's request inbox
│   ├── README.md                              ← quick reference
│   └── M{NNNN}-{slug}.md                      ← one file per request
└── logs/
    └── 미션기록/                                ← NEW: completion logs (operator-language)
        └── M{NNNN}-수행결과.md
```

Operator writes a brief into `harness/missions/M{NNNN}-{slug}.md`, then says
**"M{NNNN} 수행해"**. Tamer reads the file, dispatches to the matching
specialist(s), executes, and writes a completion log in
`harness/logs/미션기록/` — **in the language the operator wrote the brief in**.

## Triggers added (owner: tamer)

| Trigger | Action |
|---|---|
| `M\d{4} 수행해` / `진행해` / `실행해` / `run mission M\d{4}` | Read mission, dispatch to specialists, write completion log |
| `M\d{4} 마감` / `완료` | Mark mission `status: done` |
| `M\d{4} 취소` | Mark mission `status: cancelled` (with rationale) |
| `미션 목록` / `list missions` | Summarize `missions/` by status |

## Language policy — mission-scope override

The harness-wide convention is "all artifacts in English" (per
`memory/feedback_harness_language_policy.md`). The missions subsystem
**overrides** that locally:

| Artifact | Language |
|---|---|
| Mission file body | Operator's choice (whatever they wrote it in) |
| Completion log (`harness/logs/미션기록/*.md`) | **Match the mission's `language` field** |
| Mission filename slug | English (kebab-case, for tooling safety) |
| All other harness artifacts | English (unchanged) |

Rationale: completion logs are operator-facing. Forcing them into English
when the operator wrote the brief in Korean adds friction for no audit
benefit. Internal harness artifacts stay English so specialists and
external tooling don't fragment.

## Migration / usage

For a Korean-speaking operator working in this harness:

1. Write a brief into `harness/missions/M0042-something.md` (slug in
   English, body in Korean if you prefer):
   ```markdown
   ---
   id: M0042
   title: 코드 리뷰 부탁
   operator: psmon
   language: ko
   status: inbox
   created: 2026-05-10
   ---
   # 요청
   …
   ```
2. Say to Claude: **"M0042 수행해"**
3. Tamer reads, dispatches, executes, writes the completion log in Korean.

For an English-speaking operator:
- `language: en` in frontmatter, brief in English.
- Tamer dispatches identically; completion log is in English.
