## Why this change

Mission M0024 ("MS VibeVoice 영입") asked for speaker diarization in the
voice subsystem. After research (logged in the mission record), the
operator-confirmed implementation path was **Sherpa-ONNX + pyannote
v3.0 segmentation + 3D-Speaker embedding** rather than VibeVoice — the
7B VibeVoice-ASR has no official ONNX export and doesn't fit the
operator's "embedded, no pip, like the Music model" constraint. Sherpa
hits the same capability at ~50 MB vs ~5 GB.

That introduces three concerns not previously present:

1. **A diarization model pair with a specific contract** —
   segmentation+embedding+clustering instead of a single classifier
   model. Wrong pair = silent DER degradation.
2. **A transcript-merging step** — Whisper text spans + Sherpa speaker
   spans must combine via interval-overlap arithmetic with documented
   edge cases.
3. **A new PCM source for Voice test** (operator-requested) — WASAPI
   loopback parity with the Music tab, reusing
   `AgentZeroWpf.Services.Music.LoopbackCaptureService` rather than
   duplicating capture code.

None of the existing specialists own those cleanly:
- `code-coach` covers general C# / WPF / Akka but doesn't know
  diarization model pairing or AHC clustering.
- `music-curator` covers audio classification (instrument / genre)
  — a different concern from speech speaker assignment.
- `test-sentinel` owns voice round-trip testing (STT/TTS quality)
  but not the diarization merge edge cases.

Hence `voice-curator` — a dedicated expert with the Sherpa-ONNX serving
contract and the merge-pipeline conventions in one place.

## Rollback procedure

If `voice-curator` proves redundant (e.g. diarization gets shelved and
`code-coach` could absorb the small amount of knowledge):

1. Revert `harness/harness.config.json` to v1.8.0.
2. Delete `harness/agents/voice-curator.md`.
3. Move (don't delete) the two knowledge files to
   `harness/knowledge/code-coach/` and rename headers' owner field.
4. Add a v1.9.1 doc explaining the consolidation.

The knowledge files survive whichever agent ends up owning them.
