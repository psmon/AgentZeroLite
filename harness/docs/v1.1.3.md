# v1.1.3 — Agent Origin reference protocol

**Date**: 2026-04-27
**Bump**: patch (knowledge + agent triggers, no engine/agent count change)

## What changed

### New knowledge document
- **`harness/knowledge/agent-origin-reference.md`** — Defines the protocol for
  consulting `Docs/agent-origin/` (the AgentWin snapshot) instead of crawling
  the Origin codebase from scratch when the user mentions "오리진",
  "AgentWin", "조상 프로젝트", "compare with origin", etc.
- Documents lookup order, snapshot freshness policy (6-month refresh trigger),
  owner agents (`tamer`, `code-coach`), and anti-patterns.

### Agent updates
- **`harness/agents/tamer.md`**
  - Added 6 new triggers: "오리진 비교해", "오리진 참고해", "오리진이랑 비교",
    "조상 프로젝트 비교", "AgentWin 비교", "오리진 스냅샷 갱신"
  - New procedure section "오리진 비교 (오리진 비교해 / 오리진 참고해 /
    AgentWin 비교)" with 5-step lookup → refresh → log flow.
- **`harness/agents/code-coach.md`**
  - Added `agent-origin-reference.md` to "Owned convention sets" list.
  - Mode 3 (Research consult) now must cite the relevant
    `Docs/agent-origin/0[1-3]-*.md` section if a similar problem already has
    an Origin solution.

### Project-level docs
- **`CLAUDE.md`** — New section "Ancestor reference — AgentWin (Origin)"
  added at the end of the file. Tells every Claude Code session to read
  `Docs/agent-origin/` first when Origin is mentioned, and to update the
  snapshot in place when stale.

### Memory
- **`memory/reference_agent_origin_docs.md`** — Pinned reference memory so the
  protocol survives across sessions (mirrors the knowledge document for the
  user-level memory system).

## Why

The previous turn produced a 4-document AgentWin↔Lite comparison set under
`Docs/agent-origin/`. Without a binding protocol, future sessions would
re-survey the Origin codebase from scratch every time the user asked about
ancestor patterns — wasting context and risking drift from the curated
snapshot. v1.1.3 makes the snapshot the canonical entry point.

1. Remove the "Ancestor reference" section from `CLAUDE.md`
2. Drop `harness/knowledge/agent-origin-reference.md`
3. Strip the 6 origin-comparison triggers from `tamer.md`
4. Remove the `agent-origin-reference.md` bullet from `code-coach.md`
5. Delete `memory/reference_agent_origin_docs.md` and remove its line from
   `MEMORY.md`
