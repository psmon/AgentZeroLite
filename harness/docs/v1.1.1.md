# v1.1.1 — Crash Dump Forensics Ownership

**Date**: 2026-04-25
**Bump reason**: Patch — knowledge added + agent scope extended (no new agents, no new engines).

## Designated owner

Crash dump forensics (`*.stackdump`, `*.dmp`, `core`, `crash-*.log`) — primary
ownership assigned to **`security-guard`**, with `build-doctor` as secondary
correlator for build-environment context.

Rationale: crash dumps are memory-forensic artifacts that can leak credentials,
paths, and module layout, and recurring dumps can be exploitation signals — fits the
security mission. The build-time correlation matters but is a narrower lens than
the forensic primary.
