## 결과

### A. Uninstall 인프라 (이번 미션이 함께 요청한 추가 기능)

- `WebDevPluginInstaller.Uninstall(pluginId)` 신규 — staging dir 패턴 없이
  곧바로 mounted 폴더 삭제. 단, 안전 가드 두 개:
  - 입력 id 검증 + resolved 경로가 `pluginsRoot` 아래인지 prefix 검사
    (`../` 이스케이프 차단).
  - 디렉토리 미존재 시 명시 에러 ("plugin not installed").
- `WebDevPagePanel.xaml` — sample list `DataTemplate` 안에
  `Visibility="Collapsed"` 시작의 `×` 버튼 추가. `IsBuiltIn=False` 일 때만
  Visible 로 전환 (DataTrigger). 빌트인은 클릭 자체 노출되지 않음.
- `WebDevPagePanel.xaml.cs` `OnUninstallClick` — 다음 순서로 동작:
  1. `MessageBox.Show` confirm (Yes/No, 기본 No).
  2. 캐시된 WebView2 + bridge 을 먼저 detach/dispose 하고 `viewHost` 에서
     제거. 활성 view 였으면 `_activeView = null`. 이게 먼저 일어나야 핸들이
     릴리즈되어 `Directory.Delete` 가 안전.
  3. `WebDevPluginInstaller.Uninstall` 호출.
  4. 결과 보고 후 `RefreshSampleList(preserveSelection: false)` — 새 default
     (voice-test 또는 첫 항목) 로 자동 전환.
  5. `e.Handled = true` — 클릭이 부모 ListBoxItem 으로 bubble 되어 selection
     이 발화하지 않게.

### B. 음성노트 플러그인 본체 (`Project/Plugins/voice-note/`)

- **manifest.json** — `id: voice-note`, `entry: index.html`, `icon: 🎙`,
  `version: 0.1.0`. M0007 의 PluginManifest 검증 통과 형태.
- **index.html** — shell. 좌측 노트 목록 (`<aside class="vn-side">`) +
  우측 활성 노트 (`<main class="vn-main">`). 컨트롤 바 (REC / Pause /
  Sensitivity 슬라이더 / Summarize / Delete), 3-tab 본문 (Raw timeline /
  Summary / Meta), 우측 status 행. `<script src="https://zero.local/common/zero-bridge.js">`
  로 공용 brigde 로드 — 빌트인 가상 호스트 활용.
- **voice-note.css** — 다크 테마. WebDev 캔버스와 같은 `#0A0A14` 배경.
  REC 버튼은 stop 상태에서 빨간 fill, recording 일 때 점이 펄스 애니메이션.
  Sensitivity 슬라이더 `accent-color: var(--accent)`. 노트 항목 active 시
  파란 fill + REC 태그 inverse.
- **voice-note.js** — 핵심 로직 (~370 lines). 구조:
  - **IndexedDB wrapper** (`openDb` / `dbAll` / `dbPut` / `dbDelete`) — store
    `notes` (keyPath `id`), `byUpdatedAt` index. DB 이름 `voice-note-db`.
  - **state** — `notes[]`, `activeId`, `capturing`, `paused`, `sensitivity`,
    `captureStartedMs`, `activeTab`, `busy`, `transcribing`.
  - **Bridge subscription** — `bindBridge()` 에서 `note.onTranscript` /
    `note.onUtteranceStart/End` / `note.onError` 4개 구독. transcript 도착
    시 active 노트의 `raw[]` 에 `{ts, text}` push + debounced persist +
    list count 갱신 + status "captured · N lines".
  - **Capture lifecycle** — `startCapture` 가 `zero.note.start(sensitivity)`
    호출 + 활성 노트 `meta.startedAt` 기록 + 캡처 시작 시각 저장. `stopCapture`
    는 unbind + `meta.endedAt`. `togglePause` 가 `zero.note.pause/resume`.
  - **Sensitivity** — 슬라이더 조작은 즉시 `setSensitivity(value)` 로
    실시간 반영 (capture 중일 때만 wire).
  - **Note ops** — `newNote` / `selectNote` (capturing 중이면 confirm) /
    `deleteActive` (confirm). `persist` 는 400ms debounced — 빠른 입력
    (제목 타이핑) 이 IDB 폭주 안 함.
  - **Summarize** — `zero.summarize(rawJoined, 6000)` 호출. 응답 `{ok,
    summary, inputChars, chunks}` 로 노트 `summary` + `meta.summarizedAt /
    summarizedInputChars / summarizedChunks` 기록 후 자동 Summary 탭 전환.
  - **3-tab 렌더** — Raw timeline (각 발화 `mm:ss + 텍스트` 로우), Summary
    (preformatted), Meta (JSON 들여쓰기 출력).
  - **Boot** — IDB 로드 → 가장 최근 updated 노트 자동 선택 → render.
  - **beforeunload** — capture 중 페이지 이탈 시 `note.stop()` 시도해 mic
    릴리즈.

**알려진 한계 / 후속 정리 후보**:
- voice-note 가 capture 시작 시 STT provider 가 `Off` 면 `NoteError` 발생
  → 사용자에게 status 띄움 ("STT provider is Off — pick one in
  Settings/Voice first"). 안내는 명확하나 "Settings 로 바로 이동" 링크는
  웹 sub-app 에선 어렵다. 후속 미션에서 host 측 `openSettings('voice')`
  같은 deep-link op 가 있으면 UX 개선.

  가면 git installer 의 200 file cap 에 걸릴 가능성. 음성노트는 4 파일이라
  여유.
- IndexedDB 만 사용 → 다른 머신과 sync X. 후속 미션에서 export/import
