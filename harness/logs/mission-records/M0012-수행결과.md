# 수행 요약

M0011 의 token-remaining 위젯에서 모델별 그룹이 의미 없음을 사전조사로 확정
(rate_limits 가 계정 단위 quota — Opus / Sonnet 동일값 누적). 두 단계로 분할
설계 후 코드 진입.

- **Phase 1** (UX-only, 무위험): 위젯을 계정 단일 row 로 collapse, Settings 의
  per-model 토글 섹션 제거. 스키마 / wrapper / collector 무변경.
- **Phase 2** (스키마 + wrapper + UI 추가): 신규 `SessionHeartbeat` 테이블 +
  EF migration, wrapper v3.0 (session_id / cwd / project_dir 캡처 + per-session
  snapshot subdir), 신규 `SessionHeartbeatCollector` (UPSERT 의미론), 신규
  RPC + zero-bridge helper, Active Session 패널 UI (ON/OFF 토글, 1/3/5/10/15
  min window, click-to-copy session id, basename project, age 컬러 코딩).

### 6) Phase 2 — Heartbeat collector

`Project/ZeroCommon/Telemetry/SessionHeartbeatCollector.cs` (singleton):

- 7s stagger (TokenRemainingCollector 의 5s 와 1초 어긋남) + 15s cadence.
- 매 tick: (1) `LastSeenUtc < UtcNow - 1day` row prune → (2)
  `cc-hud-snapshots/**/*.json` 재귀 스캔, root 평탄 파일은 skip
  (TokenRemainingCollector 가 처리).
- UPSERT 로직: (AccountKey, SessionId) 로 기존 row lookup. 없으면 INSERT (FirstSeenUtc =

## 비고

- **EF migration `--no-build`** — `dotnet ef migrations add ... --no-build` 는 변경된
  엔티티를 컴파일 안 하므로 빈 migration 을 생성. **반드시 빌드 포함**으로 실행할
  것 (knowledge 후보).
- **Pencil ID 매핑 mistake** — 5 min window / age 분포 갱신 시 `mICPT` 가 SESS 컬럼
  ("f001…") 인데 AGE 로 착각해 "4m 32s" 빨강을 입혔음. batch_get 으로 노드 좌표
  (x=340 vs x=418) 확인해 정정. **U() 시 binding 결과 ID 와 의미를 항상 재확인**할 것.
- **Pencil 자동저장 부재** — `open_document` / `batch_design` 변경은 메모리에만 있고
  디스크 저장은 운영자가 Pencil GUI 에서 Ctrl+S 해야 함 (M0011 와 동일 제약).
  운영자에게 명시적으로 알림.
- **Wrapper v3 자동 배포** — `EnsureWrapperOnDisk()` 를 OnStartup 에 hook 했으므로
  v2 사용자도 앱 재시작만으로 v3 으로 자동 전환. 운영자 측 설정 손작업 불필요.
