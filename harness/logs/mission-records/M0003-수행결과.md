# 수행 요약

operator(psmon)의 한국어 메타 요청 — "어머니가 냉장고에 휴대폰 개통 메모를 포스트잇으로 붙이시던 그
인터페이스를, 미션 게시판이 아닌 **냉장고에 자연스럽게 붙은 포스트잇**으로 시각 개편하라" — 을
받아 수행. operator가 명시한 "design-first" 절차를 그대로 따랐다:

1. 펜슬에 디자인 초안 작성 (`Docs/design/design.pen`)
2. operator 검토 요청 / "승인 진행해" 응답 후 구현 진입
3. 폰트 확보 여부 확인 (operator 발견 — Caveat이 한글 미지원 / `Noto Sans KR`만 시스템에 있음)
4. Google Fonts CDN(Caveat + Gaegu) 채택 합의 후 구현
5. Playwright로 list view / detail modal / result tab / 필터 / 모바일 뷰포트 검증

상세 카드 내부 표현은 operator 지침대로 **기존 missions.js의 spec 카드 + md-viewer 그대로 재사용**
— 새로 만든 건 carrier로서의 모달 컨테이너와 진입/이탈 transition 뿐.

### 3. missions.js 전면 재작성 (Home/harness-view/js/views/missions.js)

기존 `todo-row` 체크리스트 → 냉장고 + 포스트잇 그리드로 교체. 라우트 호환:

| Route | 동작 |
|---|---|
| `#missions` | 냉장고만 — 모달 닫힘 |
| `#missions/{id}` | 냉장고 + PRD 탭 모달 |
| `#missions/{id}/result` | 냉장고 + Result 탭 모달 |

설계 결정:

- **모달은 detail view를 push 하는 게 아니라 fridge 위에 layer로 띄움**. params 유무로
  분기 — `render(ctx)`는 항상 fridge를 그리고, params가 있으면 `openMissionModal()`이
  `#modal-root`(기존 인프라, pen-viewer.js에서 사용)에 모달 뼈대를 mount → 마크다운을
  async load해서 끼워 넣음. 닫기는 `location.hash = '#missions'`로 — re-render에서
  params 없으면 `closeModal()`이 호출돼 modal-root가 비워짐. URL 기반 round-trip이라
  뒤로가기 / 새로고침 / 딥링크 모두 자연스럽게 작동.

- **포스트잇 색상 / 회전 / 모서리는 `M{NNNN}`의 hash로 결정** → 같은 미션은 reload해도
  같은 포스트잇 모양으로 떨어짐 (재현성). 색상 팔레트는 status 바스킷 안에서 1~2종
  variants 중 하나를 선택, 회전은 8단계 lookup 테이블, 모서리는 3종(`corner-tl/tr/bl`).

- **In Progress 포스트잇은 sway + pulse 두 애니메이션 동시 적용**. CSS의 `animation`
  속성에 두 keyframe을 컴마로 나란히 두는 정석 방식.

- **`+` Pin slot**은 클릭 가능하지 않은 정적 안내. 미션 등록은 in-app 기능이 아니라 디스크
  파일 작성 + 인덱스 재빌드 흐름이라, 그 사실을 시각적으로 노출만 한다(`harness/missions/M{NNNN}-*.md`).

- **접근성**: 포스트잇은 `role="button"` + `tabindex="0"` + Enter/Space 키 활성화. 모달은
  ESC 닫기 + 백드롭 클릭 닫기. `prefers-reduced-motion: reduce` 메디아쿼리 체크 — sway /
  pulse / pop 모든 애니메이션 비활성화.

### 펜슬 라이브 렌더 캐시 이슈 (작업 중 발견)

`mcp__pencil__get_screenshot`이 새로 insert/copy된 노드에 대해 **빈 화면**을 반환하는 현상.
디버그 노드(`#FF0000` 단순 사각형)도 동일 — 즉, 데이터는 정상 저장되지만 라이브 화면 캐시가
즉시 갱신되지 않음. `mcp__pencil__open_document`로 reload하면 in-memory 변경분이 디스크와
재동기화되며 일부가 사라지는 부수 효과까지 있어 원래 디자인을 한 번 다시 만들었다.

해결: `mcp__pencil__export_nodes`로 PNG export — 이건 정확히 렌더된다. 차후 펜슬에서 즉각
검증이 필요하면 screenshot보다 export 경로를 우선 시도할 가치 있음. `references/` 노트로
정리해 둘 만한 lesson.

### 포스트잇이 Playwright 클릭에 비협조적인 이유

CSS sway 애니메이션이 무한 반복돼서 Playwright의 "element is visible, enabled and stable"
체크가 5초 내내 false로 떨어진다. 실제 사용자 마우스는 stability check를 하지 않아 정상
동작 — 즉 e2e 테스트의 인공물이지 실제 UX 결함은 아니다. 향후 e2e 자동화가 필요하면:

- `:hover`/`:focus`로 sway가 멈추는 지점에서 클릭 (페이지에서 `mousemove` 후 클릭)
- 또는 테스트 모드 플래그로 sway 비활성화
- 또는 `prefers-reduced-motion` 시뮬레이션

본 미션에선 hash navigation으로 우회 — 라우트 자체는 항상 동작하므로 e2e coverage 손실 없음.

### 일부러 안 한 것 — 드래그로 포스트잇 이동

operator는 `(TBD)`로 표시한 "Drag to rearrange" 힌트만 펜슬에 둔 상태. 실제로 구현하려면:
1. 위치를 어딘가에 영속화(localStorage? indexes에 metadata?)
2. 드래그 중인 포스트잇의 sway 비활성화 / z-index 부상
3. drop 위치 검증 + 충돌 회피

이는 본 미션 범위 밖. UI에 안내 텍스트로 의도만 노출(`hint: 'Tap a sticky note to open
its details. Drag to rearrange (TBD).'`) — 별도 미션으로 분리되면 자연스럽게 후속.

### 일부러 안 한 것 — md-viewer breadcrumb 중복

모달 안에 들어간 md-viewer가 자체 breadcrumb(`harness / missions / M0003-...md` + Reference
배지)를 그린다. 모달 헤더에 이미 path를 보여줘서 시각적으로 중복인데, 이는 md-viewer가
다른 4~5개 view에서도 공유하는 컴포넌트라 "missions에서만 빼기"는 view-별 prop drilling을
유발한다. 실용적 영향이 적은 cosmetic 중복이므로 판단을 미뤘다 — md-viewer의 `breadcrumb`
prop을 명시적으로 빈 배열로 넘기는 옵션을 추가하면 한 줄로 해결되지만, 다른 사용처와의
default 호환성을 고려해 별도 미션으로 다룰 만하다.

### 일부러 안 한 것 — 셀프호스트 폰트 woff2

Google Fonts CDN으로 갔다. 합의된 결정이지만 트레이드오프 메모:

- **CDN 장점**: index.html 1줄, 빌드 의존성 없음, 캐시 공유 효과 가능
- **CDN 단점**: 오프라인 환경 폴백(`cursive`)으로 전환, Google에 ping 발생, 네트워크 첫
  로드 지연
- **셀프호스트 woff2 장점**: 외부 의존 0, 일관된 렌더, 프라이버시
- **셀프호스트 woff2 단점**: ~120KB 번들 증가, 빌드에 폰트 디렉토리 추가, 4개 파일 관리

기존 marked / mermaid / lucide도 CDN인 일관성을 따랐다. 오프라인 deploy 시나리오가 등장하면
woff2로 swap (15분 작업).
