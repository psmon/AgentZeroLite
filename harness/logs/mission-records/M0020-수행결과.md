# 수행 요약

Supertone Inc 의 **SuperTonic** (`pip install supertonic`) 을 AgentZero Lite 의
4번째 TTS 제공자로 영입했습니다. 이는 **AgentZero가 사용자 환경에 설치된
Python pip 패키지를 서브프로세스로 호출하는 첫 사례** 입니다. 기존 3 종(`Off`,
`WindowsTTS`, `OpenAITTS`) 옆에 `Supertonic` 옵션이 Voice 탭에 추가되었고,
사용자가 선택했을 때만 Python/언어/Steps 필드와 **"Check Install"** 프로브가
나타납니다.

## 비고

### 의도적으로 보류한 항목 (후속 미션 후보)

1. **모델 자동 다운로드 진행률 UI**.
   첫 합성 시 ~400 MB 가 `~/.cache/supertonic3/` 로 받아지는데, 현재는 그
   다운로드가 supertonic CLI 내부에서 일어나므로 진행률을 가로채려면 stderr
   라인을 파싱하거나 캐시 디렉토리 크기를 폴링해야 합니다. 1차에서는 Probe
   로 설치 여부만 확인하고, 모델 다운로드는 첫 합성 호출이 단순히 길어지는
   것으로 처리. M0017 의 Whisper 모델 다운로드 진행률 UI 와 같은 수준으로
   끌어올리려면 별도 미션이 필요합니다.

2. **`pip install supertonic` 자동 실행**.
   현재는 사용자에게 명령어를 안내만 합니다. 자동 실행은 (a) 권한 있는 Python
   환경 선택 UX, (b) venv 권장 흐름, (c) PEP 668 환경에서의 break-system-packages
   처리 등 검토할 게 많아 1차 범위에서 제외. M0014 (OS 자동화) 와 합쳐 다룰
   여지가 있습니다.

3. **OpenRAIL-M 라이선스 고지 다이얼로그**.
   현재는 Settings 패널 안에 짧은 안내만 표시합니다. 상업 배포 시 use-based
   restrictions (딥페이크/사칭/허위정보 금지) 를 사용자가 명시적으로 동의하게
   하는 모달이 필요하다면 후속 작업으로 추가.

4. **Akka 스트림 파이프라인 통합**.
   `VoiceStreamActor` 의 `TtsWorkerActor` 풀에서도 Supertonic 이 자동으로
   라우팅되도록 했지만 (BuildTts 가 공유 팩토리이므로), Sentence chunk 단위로
   짧은 합성을 반복할 때 매 호출이 새 Python 프로세스를 띄우는 비용이 무시
   못할 수 있습니다 (~150ms ColdStart). 후속에서 한 번 띄운 supertonic 데몬
   프로세스에 stdin 으로 텍스트를 흘리는 패턴 도입 고려.

### Python.NET 임베딩을 고른 길이 아닌 이유

조사 단계에서 `Python.NET` / `IronPython` 으로 supertonic 을 in-process 임베딩
하는 안도 검토했지만 거절했습니다:

- Python.NET 은 CPython 런타임을 .NET 프로세스에 고정해 게시본/Inno 설치 패키지가
  복잡해지고 RID 분기가 늘어남.
- IronPython 은 ONNX Runtime 휠과 호환되지 않음 (CPython native extension 의존).
- supertonic CLI 의 RTF 0.012~0.015 (M4 Pro 기준) 이면 짧은 발화 (<100ms 합성)
  에 대해서도 ~50ms 의 Python 프로세스 콜드 스타트 비용을 충분히 흡수.

서브프로세스 호출로 가는 게 .NET 10 게시본을 깔끔하게 유지하면서 사용자가
Python 환경을 자기 venv 로 분리할 수 있게 해 줍니다.

### 사용 흐름 (사용자 관점)

1. Settings → Voice → TTS Provider 콤보에서 "Supertonic" 선택.
2. Python 입력은 비워두면 PATH 의 `python` 사용. 별도 venv 가 있다면 절대 경로 입력.
3. **Check Install** 클릭 → `tbTtsStatus` 에 `✓ Supertonic ready.` 또는 `✗ Supertonic not found. Run: pip install supertonic` 표시.
4. 미설치 안내가 뜨면 사용자가 직접 CMD/PowerShell 에서 `pip install supertonic` 실행.
5. Language 콤보 (Korean ko / English en / Japanese ja / Chinese zh / Auto na) 선택, Steps (5..12) 조정.
6. **Refresh** 클릭 → `cbTtsVoice` 에 M1..M5 / F1..F5 10종 자동 채움. 원하는 보이스 선택.
7. **Save TTS** 클릭.
8. Voice Test 시작 → STT/LLM/TTS 풀체인이 동일 파이프라인을 타며 Supertonic 이 WAV 를 생성하면 NAudio 가 재생.

### 진단

PATH 캐시 문제가 정답. Windows 프로세스는 시작 시점에 부모(보통 explorer.exe)에서
PATH 를 상속받고, 이후 시스템 PATH 가 변경돼도 **이미 떠 있는 프로세스에는
반영되지 않는다**. AgentZero 가 Python 3.12 설치보다 먼저 떠 있었다면
`Process.Start("python")` 은 옛 PATH 에서 `python` 을 찾기 때문에:

- 못 찾으면 `Win32Exception` (1차 구현은 "Probe failed: …" 로 표시 — OK)
- Windows Store stub (`%LOCALAPPDATA%\Microsoft\WindowsApps\python.exe`) 에 걸리면
  exit code 9009 으로 조용히 실패 → 1차 구현은 이걸 **"Supertonic not installed"
  로 잘못 표시** (실은 Python 자체가 잘못된 인터프리터). 진단이 거의 불가능.

옆 터미널은 새로 떠서 갱신된 PATH 를 받았고, 그래서 `pip install` 이 정상
`C:\Users\psmon\AppData\Local\Programs\Python\Python312\python.exe` 로 동작.

### 수정

`SuperTonicTts.EnsureReadyAsync` 를 **2-phase 프로브** 로 재작성:

- **Phase 1**: `{python} -V` — 인터프리터 자체가 해상되는지 + 어떤 버전인지 확인.
  실패 시 메시지: `"Python not reachable as 'python'. If you just installed
  Python, restart AgentZero so PATH refreshes, or paste the full path (e.g.
  C:\Users\<you>\AppData\Local\Programs\Python\Python312\python.exe)."`
- **Phase 2**: `{python} -m pip show supertonic` — 같은 인터프리터의 site-packages
  에 패키지가 있는지 확인. 실패 시 메시지: `"{Python version} found, but
  supertonic is NOT installed in this interpreter. Run: {python} -m pip install
  supertonic"` — 어떤 Python 에 설치해야 하는지 명시.
- 성공 시 메시지에 **Python 버전 + Supertonic 버전 둘 다** 표시 — 사용자가
  "어떤 Python 에 깔렸나" 를 한 줄로 확인 가능.

부수 효과:
- `python -V` 가 stdout 대신 stderr 로 출력하는 구식/quirky 빌드도 허용.
- `Process.Start` 자체가 throw 하는 경우 (잘못된 경로) 메시지에 명시적으로
  "restart AgentZero or set full path" 안내.

## 후속 수정 #5 — `python -m supertonic` 폐기, 라이브러리 API 로 전환

### Operator 보고 (verbatim)

> 다음 오류 재생이안됨..
> [13:46:35] tts failed: supertonic exited with code 1.
> stderr: C:\Users\psmon\AppData\Local\Programs\Python\Python312\python.exe:
>   No module named supertonic.__main__;
>   'supertonic' is a package and cannot be directly executed

### 진단

1차 영입 때 `python -m supertonic tts <text> --voice F1 --lang ko --steps 8 -o out.wav`
서브프로세스 형태를 가정했는데 — `supertonic` 패키지는 **`__main__.py` 가 없다**.
PyPI 의 supertonic 1.2.3 은 CLI 를 (a) `setup.py` 의 `entry_points` →
`<Python>\Scripts\supertonic.exe` 단일 실행파일과 (b) `from supertonic import TTS`
라이브러리 API 로 노출하지, `python -m` 으로는 호출 불가.

옵션 평가:
- **CLI 실행파일 직접 호출** (`<Python>\Scripts\supertonic.exe`) — 사용자가 지정한
  python.exe 옆의 Scripts 디렉토리에서 supertonic.exe 를 찾아야 함. per-user /
  per-machine / Microsoft Store / venv / pyenv 레이아웃마다 Scripts 경로가
  미묘하게 달라서 fragile.
- **`python -c "<script>"` + 라이브러리 API** — 선택한 python.exe 가 supertonic 을
  import 할 수만 있으면 무조건 통함. 인터프리터-경로 의존성만 신경쓰면 됨 →
  후속 #2/#3/#4 의 picker 와 진단이 그대로 유효.

→ 라이브러리 API 채택.

### 수정

`SuperTonicTts.SynthesisScript` 상수에 인라인 Python 스크립트 박음:

```python
import sys
from supertonic import TTS
text, voice, lang, out, steps = sys.argv[1:6]
tts = TTS(auto_download=True)
style = tts.get_voice_style(voice_name=voice)
kwargs = dict(text=text, voice_style=style, total_steps=int(steps))
if lang and lang != 'na': kwargs['lang'] = lang
wav, _ = tts.synthesize(**kwargs)
tts.save_audio(wav, out)
```

새 argv 형태: `<python> -c <script> <text> <voice> <lang> <output> <steps>`.
사용자 텍스트는 `sys.argv[1]` 로 분리 전달 → 스크립트 본문에 임베드하지
않으므로 따옴표/줄바꿈 이스케이프 사고 없음 (Process.StartInfo.ArgumentList 가
인자별 Win32 쿼팅 처리).

Supertonic 라이브러리 API 호출 규약 (소스 검증):
- `TTS(auto_download=True)` — 첫 호출 시 ~400MB 모델을 `~/.cache/supertonic3/`
  로 자동 다운로드, 이후 캐시 재사용
- `tts.get_voice_style(voice_name="F1")` — 보이스 ID 를 style 객체로 변환 (단순
  문자열로 `synthesize` 에 못 넘김)
- `tts.synthesize(text, voice_style, lang, total_steps)` → `(wav: ndarray, duration)`
  튜플 (단일 ndarray 아님)
- `tts.save_audio(wav, out)` — soundfile 사용한 내장 WAV 저장 헬퍼 (외부 의존
  추가 불필요)

`lang=""` 또는 `lang="na"` 는 라이브러리 호출 시 kwargs 에서 제거 → 라이브러리
의 auto-detect 동작 사용.
