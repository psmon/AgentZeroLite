# 수행 요약

오리진(AgentWin) 의 OS 제어 표면을 AgentZero Lite 구조에 맞춰 흡수했다.
수락 기준은 (1) CLI 에 `os <verb>` 그룹 추가, (2) LLM agent loop 에
function-call 도구 6종 노출, (3) 입력 시뮬레이션은 게이팅, (4) 스크린샷/
audit 가 `tmp/os-cli/` 에 누적, (5) E2E 스모크 1건 + 문서 일괄 갱신.
다섯 항목 모두 충족.

오리진 사전조사 결과 (`Docs/agent-origin/03-adoption-recommendations.md`
§P1-2) 가 이미 *AgentZeroWpf.Automation* 별도 csproj 분리를 권고했으나,
실측 결과 NativeMethods.cs 와 ElementTreeScanner 가 이미
`Project/AgentZeroWpf/` 안에 자리잡고 있어 폴더 분리 (`OsControl/`)
만으로 충분했다. 분리 csproj 는 *premature*.

```
AgentZeroLite.exe -cli os help          → 사용법

AgentZeroLite.exe -cli os mouse-wheel <x> <y> <delta> [--allow-input]

AgentZeroLite.exe -cli os audit [--last N]
```

기존 13 개 CLI 동사에 더해 12 개 추가 → 총 25 개 표면.

### LLM 도구 표면 (6 new tools)

`AgentToolGrammar.KnownTools` 가 5 → 11 로 확장:

| 신규 tool | args | gate |
|---|---|---|
| `os_list_windows` | `{title_filter?}` | none |
| `os_screenshot` | `{hwnd, grayscale}` | none |
| `os_activate` | `{hwnd}` | none |
| `os_element_tree` | `{hwnd, depth, search?}` | none |
| `os_mouse_click` | `{x, y, right, double}` | **AGENTZERO_OS_INPUT_ALLOWED** |
| `os_key_press` | `{key}` | **AGENTZERO_OS_INPUT_ALLOWED** |

GBNF `toolname` alternation, `AgentToolGrammar.SystemPrompt` 의 도구
카탈로그 섹션, `LocalAgentLoop.ExecuteToolAsync`, `ExternalAgentLoop
.ExecuteToolAsync` 모두 동기화.

### Approval gate (입력 시뮬레이션 only)

이중 enabler — 둘 중 하나면 충분:

1. **CLI per-call**: `mouse-*` / `keypress` 동사에 `--allow-input` 부착
2. **Process env**: `AGENTZERO_OS_INPUT_ALLOWED=1`

읽기 전용 동사 (window enum, screenshot, UIA tree, DPI, activate)
는 무조건 동작. per-call 휴먼 confirm 은 없음 — 자동화 마찰을 안기면
기능 의미가 사라지기 때문. 대신 audit 로그가 사후 forensic 역할.

게이트 거부 시 응답:

```json
{
  "ok": false,
  "error": "OS input simulation is gated. Set environment variable AGENTZERO_OS_INPUT_ALLOWED=1 (LLM/GUI) or pass --allow-input on the CLI verb to enable.",
  "verb": "<tool name>"
}
```

LLM 시스템 프롬프트가 게이트 거부 시 **재시도 금지** + `done` 즉시 호출
지시 — repeat-defense 가드 (M0013, AgentLoopGuards.MaxSameCallRepeats)
와 결합돼 갇힐 가능성 차단.

### Audit 트레일

`tmp/os-cli/audit/<yyyy-MM-dd>.jsonl` 에 한 줄씩 append. CLI/LLM/E2E
삼자 호출이 모두 같은 schema. 실패한 호출도 `ok:false` 로 기록.

```json
{"ts":"2026-05-07T22:30:09.123+09:00","caller":"cli","verb":"screenshot",
 "args":{"hwnd":0,"grayscale":true,"fullDesktop":true,"path":"…\\desktop.png"},
 "ok":true,"error":null}
```

### E2E 스모크

`Docs/scripts/launch-self-smoke.ps1` — pwsh 스크립트. 단계:
GUI 살아있는지 → `os list-windows --filter "AgentZero Lite"` →
`os get-window-info <hwnd>` → `os screenshot --hwnd <hwnd>` →
`os element-tree <hwnd> --depth 3` → `os dpi`. 결과는
`tmp/os-cli/e2e/<date>.log` 에 누적.

엔진 정의: `harness/engine/os-cli-e2e-smoke.md`. operator 가
`OS CLI 스모크` / `os smoke` / `M0014 스모크` 트리거로 호출 가능.

## 비고

- ChromaKey / DPI per-monitor 정확성: `DpiHelper.Query()` 가 monitor pt 기반.
  멀티 모니터에서 커서가 다른 monitor 에 있으면 그 monitor 의 effective DPI
  를 반환. 이 결정은 의도적 — "currently active 모니터" 가 일반적으로
  스크린샷의 해석 기준이기 때문.
- `WindowEnumerator.Activate` 의 AttachThreadInput dance 는 Win10+ rate
  limit 우회용. 실패해도 best-effort — `ok:false` 로 audit 기록.
- `ScreenshotService` 의 1920×1080 다운스케일 default 는 LLM 컨텍스트 보호
  목적이 아니라 **파일 사이즈 제어**. LLM 은 어차피 PNG bytes 를 받지
  않고 파일 경로만 받음. 풀해상도 필요 시 `--full` 무시 + `--hwnd 0`
  + downscale1080=false 가 가능하지만 현재 CLI flag 로 노출 안 함.
- M0013 (AgentLoopGuards) 의 MaxSameCallRepeats=3 은 OS 도구에도 자동
  적용. 즉 LLM 이 같은 좌표로 mouse_click 을 3 회 반복하면 가드가
  걸어 자기교정 시그널을 받음. 본 미션이 별도 가드를 추가할 필요 없었음.
