# 수행 요약

Settings → WebDev 탭 안에 갇혀 있던 WebView2 sandbox를 MainWindow의 공식 1-depth
메뉴로 승격. 좌측 ActivityBar의 AgentBot 다음 자리에 globe 아이콘을 추가했고,
클릭하면 240px 샘플 리스트 + 풀-윈도우 WebView2 의 3-column 레이아웃이 열린다.
Settings의 옛 자리는 안내 튜토리얼(`WebDevTutorialPanel`)로 교체했다.

추가로, 샘플 리스트는 더 이상 ComboBox 두 개로 하드코딩되어 있지 않다. 빌트인은
`{exeDir}/Wasm/*` 디렉토리 enumerate, 사용자 플러그인은
`%LOCALAPPDATA%/AgentZeroLite/Wasm/plugins/*` 에서 `manifest.json` 기반으로
자동 발견된다. 새로운 "+ Install Plugin" 항목으로 .zip 파일을 선택하면 안전한
압축 해제 + manifest 검증 + 마운트 후 메뉴가 즉시 갱신된다.

WebDev를 단순 sandbox에서 **동적 확장 가능한 플러그인 호스트**로 끌어올린 것이
이번 mission의 본질적 변화.

## 후속 수정 #3 (operator 3차 검수 — 샘플 전환 cold-load 제거)

operator 보고: **WebDev 하위 요소(샘플) 전환 시 매번 로드되어 느리다. 한 번
로드된 것은 유지되어야 한다 — Reload 버튼으로 명시 갱신만 하면 됨**.

원인: 단일 `WebView2` 인스턴스에 매 selection 마다 `Navigate(url)` 호출 →
WebView2 가 페이지를 cold load. JS 상태도 매번 reset.

**수정** (`WebDevPagePanel.xaml.cs` 전면 재작성):
- 단일 `webView` → `viewHost` Grid 컨테이너로 변경
- `Dictionary<string, ViewSlot> _viewsBySampleId` — sample id 별 WebView2 +
  WebDevBridge 페어 캐시 (record `ViewSlot(WebView2, WebDevBridge)`)
- `_host` (`WebDevHost`) 와 `_env` (`CoreWebView2Environment`) 는 **공유** —
  단일 LLM 세션 / 단일 voice pipeline / 단일 user-data folder. 플러그인이
  hop 사이에서도 일관된 상태를 본다.
- `OnSampleSelected` 흐름:
  1. cache hit → `ShowOnly(view)` 만 호출 (Visibility 토글, JS keeps running)
  2. cache miss → 새 WebView2 생성 → `viewHost.Children.Add` →
     `EnsureCoreWebView2Async(_env)` → 가상 호스트 매핑 (built-in + plugin
     양쪽) → bridge wire → Navigate → 캐시 등록 → `ShowOnly`
- `OnReloadClick` → 활성 view 만 `Reload()` (다른 캐시는 건드리지 않음)
- `OnDevToolsClick` → 활성 view DevTools open
- `OnHostWindowClosed` → 모든 캐시된 view + bridge dispose, host dispose

부수 효과 / 보장:

## 후속 수정 #2 (operator 2차 검수 — airspace 이슈)

operator 보고: **owner-draw 윈도우(ConPTY 터미널)가 z-order를 먹고 WebDev 위에
그려진다**. 정확한 진단. WPF airspace 문제 — `HwndHost`로 박힌 native HWND
(ConPTY 터미널, 봇 패널의 WebView2 등)는 D3D 합성 위에 GDI로 직접 그리기 때문에
같은 위치의 WPF 오버레이로 가려지지 않는다.

**수정**:
- `OnActivityWebDevClick` 보강 — WebDev open 시 `CliPanel.Visibility = Collapsed`
  + `BotDockArea.Visibility = Collapsed` 로 native 컨테이너 자체를 measure pass
  에서 제거. WPF가 이걸 감지하면 `HwndHost.OnVisualParentChanged` 경유로
  underlying HWND에 `ShowWindow(SW_HIDE)` 호출 → native window 가 실제로 가려짐.
- `BotDockArea` 의 원래 visibility 는 `_botDockVisBeforeWebDev` 에 캐시했다가
  닫을 때 복원 (사용자가 봇을 켜뒀는지 꺼뒀는지 보존).
- `CloseWebDev()` helper 분리 — 모든 WebDev 종료 경로(아이콘 토글, Settings
  진입, `SwitchToCliPanel`)에서 같은 정리 코드 호출. Idempotent.
- `SwitchToCliPanel` + `OnSidebarSettingsClick` 진입부에 `CloseWebDev()` 호출
  추가 — WebDev open 상태에서 Settings/CLI 로 갈 때 z-order 충돌 방지.

빌드 재검증: **0 errors**.

이전 1차 후속 수정 (Content Area → 풀 오버레이) 와 합쳐서 WebDev 는 이제:
1. ActivityBar(48px) 만 살리고 나머지 영역 풀 점유 (XAML span 으로 layout 보장)

## 후속 수정 #1 (operator 1차 검수 피드백)

operator 1차 검수에서 다음 문제 보고: **WebDev가 workspace의 하위 영역에서만
표시됨 — 우측 패널을 다 사용하지 못하고 SidePanel(workspaces) + 하단 BotDockArea
가 같이 보임**. 디자인은 풀-폭 레이아웃이었는데 구현은 Content Area Grid(Column 3)
안에 박혀 있어 좁아 보였다.

**수정**:
- `WebDevPagePanel`을 Content Area Grid 안에서 빼서 메인 Sidebar+Content Grid
  의 마지막 자식으로 이동. `Grid.Column="1" Grid.ColumnSpan="3" Grid.RowSpan="3"`
  로 SidePanel + topSplitter + Content + botDockSplitter + BotDockArea 전 영역
  을 풀-오버레이로 덮음. ActivityBar(Column 0)만 살아있다.
- `OnActivityWebDevClick` 단순화 — 더 이상 CliPanel/SettingsPanel을 건드리지
  않는다 (위에서 풀-오버레이가 덮으니까). 토글만 하면 닫을 때 사용자가 이전에
  보던 CLI/Settings/Bot 상태가 그대로 복원된다.
- `SwitchToCliPanel`에서 `WebDevPage.Visibility = Collapsed` 라인 제거 — 위
  단순화로 인해 불필요.
