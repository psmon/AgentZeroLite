## 후속 수정 #2 — Scroll capture 폴백 + live preview

### Operator 보고 verbatim
> "스크랩 프리뷰가 없으며 스크롤이 작동안되 인피니티 스크롤 스크랩 불가.. 로그확인해 개선"

### 진단 (root cause)
앱 로그 (`bin/Debug/.../logs/app-log.txt`) 분석:

```
[20:04:43] [Capture] 시작 | hwnd=0x002E152A, class="SunAwtFrame", title="IntelliJ..."
[20:04:43] [Capture] Strategy 1: TextPattern → 실패
[20:04:43] [Capture] Strategy 2: FocusedAreaCapture
[20:04:43] [FocusedArea] 스크롤 가능 자식 영역 없음   ← UIA 가 Swing 컴포넌트를 못 봄
[20:04:43] [Capture] FocusedAreaCapture 실패
[20:04:43] [Capture] Strategy 3: ScrollPattern + TreeWalk
[20:04:43] [Capture] ScrollPattern 성공 | 82자        ← 윈도우 타이틀 + "System" 만
```

scrap 파일 (`logs/scrap/2026-05-16-20-04-43-scrap.txt`, 92 bytes) 내용:
```
AgentZeroLite – D:\Code\AI\AgentZeroLite\harness\missions\M0019-WIN스크랩툴.md
System
```

**두 증상의 단일 root cause** — Swing/AWT (`SunAwtFrame`) / 일부 웹뷰 / 일부 IDE 는
**UIA `ScrollPatternAvailable` 속성을 어떤 자식 요소에도 노출하지 않는다**. 그래서:
- Strategy 2 (auto-scroll 캡처) 의 `FindAll(IsScrollPatternAvailableProperty)` 가 0건 →
  스크롤 자체가 시도되지 않음 → "스크롤이 작동안되".
- Strategy 3 (static tree walk) 는 *스크롤 없이* UIA 트리만 dump → 화면에 보이는
  최소한의 root-level 요소만 잡힘 → 82자 → "프리뷰가 없는 것 처럼 보임".

### 수정

**Strategy 2.5 — `TryClipboardScrollCapture`** 신설. Strategy 2 (UIA scroll) 와
Strategy 3 (tree walk) 사이에 끼워넣음. 의존성은 키보드만 — `Ctrl+A/C/Home/PgDn`
를 받는 모든 앱에서 동작:

```
1. SetForegroundWindow + 윈도우 중앙 클릭 (포커스 확보)
2. Ctrl+Home (문서 맨 위)
3. 초기 Ctrl+A → Ctrl+C → 클립보드 읽기 → scrap.WriteAll  ← 첫 chunk = preview 즉시 채움
4. 루프 (max = scroll.MaxAttempts, default 500):
   a. PageDown
   b. Ctrl+A → Ctrl+C → 클립보드 읽기
   c. 길이가 늘었으면 새 tail 만 scrap.WriteAll(diff) → preview 라이브 추가
   d. 3회 연속 진전 없으면 break
5. finally: 원본 클립보드 복원
```

핵심 설계:
- **새 chunk 만 emit** — `next.StartsWith(accumulated)` 이면 tail 만 ChunkWritten 으로
  발사. 그러므로 preview pane 은 *덧셈*만 받고 화면이 깜빡이지 않음.
- **원본 클립보드 복원** — 사용자가 캡처 전에 가지고 있던 클립보드 내용을 finally
  단계에서 `System.Windows.Clipboard.SetText(originalClipboard)` 로 되돌림. 캡처가
  사용자의 일상 워크플로우 (복사한 텍스트) 를 침해하지 않음.
- **OperationCanceledException 그대로 throw** — 사용자가 STOP 누르면 즉시 종료
  + 그 시점까지 누적된 결과 반환.
- 이미 존재하는 helper (`SimulateSelectAllCopy`, `ReadClipboardText`) 와
  `NativeMethods.VK_HOME / VK_NEXT / keybd_event` 재사용 — 새 P/Invoke 0개.

**부가: ScrapPagePanel UI 측 ChunkWritten 진단 로그** — `[Scrap.UI] ChunkWritten →
AppendWithCap | +N자 (txt before=M자)` 로 각 chunk 가 preview pane 에 실제로 도달
했는지 확인 가능. 다음 smoke test 의 진단 비용을 0 으로 줄임.

### 잠재 한계 (다음 후속 수정 후보)
- **Ctrl+A 가 무거운 페이지 (예: 수만 줄 코드)** 에서 OOM 또는 freeze 가능 — 그
  경우 `MaxClipboardBytes` cap 추가 + 초과 시 PageDown 만 사용하는 모드 fallback.
- **PageDown 이 viewport 만 이동하는 앱 (chat 등)** 은 Ctrl+End 한 번 누른 뒤
  Ctrl+Home → Ctrl+A 로 *전체 내용 한 번에 잡기* 가 더 효율적. 필요 시 분기 추가.
