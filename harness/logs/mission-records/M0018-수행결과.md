# 수행 요약

Actor World 메뉴 (`#actor-world`) 의 3D 시뮬레이션 뷰 옆에 **한국어 전용 액터모델
듀토리얼** 하위 페이지를 추가했습니다. 운영 라우트:

- `#actor-world` — 기존 3D 우주정거장 시뮬레이션 (영문 UI, 변경 없음)
- `#actor-world/tutorial` — **신규** Korean tutorial MD viewer

영문 정식 레퍼런스는 [Akka.NET 공식 *What is Akka?*](https://getakka.net/articles/intro/what-is-akka.html)
를 본문 상단에서 한 줄로 안내하고 (영문 콘텐츠 중복 작성 금지), 본 듀토리얼 자체는
**같은 개념을 AgentZero Lite 가 실제로 쓰는 패턴으로 풀어쓴 한국어 버전**으로 작성.

## Learned (학습)

1. **CREATOR 가 항상 새 메뉴를 의미하지 않는다** — 기존 메뉴 뷰의 `params` 분기로
   sub-page 를 처리하면 사이드바 메뉴 폭주를 막을 수 있고, 한 도메인 (액터 시각화)
   안의 두 형태 (3D 시뮬레이션 / 듀토리얼) 가 자연스럽게 묶임. *Roles* 카드 → MD
   진입 패턴과 동일하지만 카드 그리드 없이 *씬 → MD* 로 바로 진입하는 변형.
2. **ComfyUI provider 는 background job 으로 굴리기 적합** — 10장 1024x576 (4 step)
   생성에 GPU 로 약 1분 내외. 메인 작업 (MD 작성) 과 병렬로 실행하면 wall-time 이
   tutorial-write 시간 안에 흡수됨. `IMAGE_GEN_ROOT` env 만으로 저장 위치를 자유롭게
   재지정 가능 — 미래 미션에서도 같은 패턴으로 in-repo asset 폴더 직접 채우기 가능.
3. **`loadMd('harness/...')` 가 그냥 동작한다** — Pages 가 repo 전체를 artifact 로
   올리는 구조 (CI-time mirror 폐기 후) 라, 듀토리얼 본문을 `harness/knowledge/_shared/`
   에 두는 게 가장 자연스러움. `Home/harness-view/data/` 같은 사전구현 폴더에 넣지
   않아도 됨 — 본문은 결국 markdown 콘텐츠라 knowledge 트리에 사는 게 의미적 맞음.
4. **마크다운 이미지 경로는 `../../Home/...` 패턴** — MD 가 어디에서 fetch 되든 (loadMd)
   *브라우저는 페이지 URL 기준* 으로 img src 를 해석. 페이지 URL 이
   `Home/harness-view/` 이므로 `../../` 가 repo root → 그 아래 `Home/harness-view/assets/...`
   가 동작. 이 트릭을 다른 듀토리얼에서도 재사용 가능.
5. **미션 backlog 의 § "아직 활용 못한 부분" 절은 학습 가속기** — Akka 의 미사용
   기능을 P0~P3 로 우선순위화하는 작업이 그 자체로 "다음에 무얼 해야 할지" 의 PDSA
   Act 단계를 압축해 준다. 앞으로 듀토리얼 미션 전반에 동일 패턴 (현재 / 미활용 /
   우선순위 매트릭스) 을 권장.
