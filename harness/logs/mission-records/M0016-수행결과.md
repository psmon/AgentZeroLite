- 표면: 시간 변화 노이즈 + 역할 베이스 컬러 mix

- 작은 ambient lift (15%) 로 어두운 면도 색이 보임

## 후속 수정 #1 — Three.js bare-specifier 해결 (import map 도입)

### Operator 보고 (verbatim)

> Three.js failed to load: Failed to resolve module specifier "three".
> Relative references must start with either "/", "./", or "../"..
> Check network.

### 진단 (root cause hypothesis — fix 전)

`https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js`
의 1번째 줄이:

```js
import { ... } from 'three';   // bare specifier
```

브라우저는 bare specifier 를 못 풀음 — `/`, `./`, `../`, 또는 full URL 만
허용. Three.js 0.150+ jsm 예제 파일들은 모두 이 패턴이라 **import map**
(또는 esm.sh 같은 specifier-rewriting CDN) 이 필수. 초기 구현은 두 모듈을
독립 URL 로 직접 import 했기 때문에 OrbitControls 의 internal `from 'three'`
가 폭발.

부수 위험: 두 다른 CDN 에서 Three.js 를 가져오면 module identity 가
깨져 OrbitControls 의 `instanceof Vector3` 같은 check 가 실패할 수 있음.
import map 으로 단일 경로 매핑하면 둘 다 같은 모듈 인스턴스를 공유.

### 수정 (file list)

- **`Home/harness-view/index.html`** (수정) — `<script type="importmap">`
  추가, `three` → unpkg URL, `three/addons/` → unpkg jsm 경로. 코멘트로
  "Three.js 버전 업그레이드 시 두 곳 동시 수정" 명시.
- **`Home/harness-view/js/views/actor-world.js`** (수정) — `THREE_URL` /
  `ORBIT_URL` 상수 제거, `await import('three')` + `await import('three/addons/controls/OrbitControls.js')`
  로 단순화. 실패 메시지에 "import map in index.html" 포인터 추가해 다음
  에러 발생 시 진단 가속.

### 빌드 재검증

- `node -e "..."` syntax check pass
- 인덱스 매니페스트는 영향 없음 (BUILDER 재실행 불필요 — `Home/harness-view/*`
  자체 변경은 매니페스트 대상 아님)

### 검수 포인트
