# 수행 요약

`Project/Plugins/token-remaining/` 신규 플러그인 + 호스트측 statusLine wrapper
설치/제거 + 30s 콜렉터 + DB 영속 모델별 row 의 풀스택 구현. v1 출시
직전 단계까지 도달. claude-hud chain pipe-through 의 자식 프로세스
실행 (exit=1) 한 가지 경로만 다음 iteration 으로 넘김.

## 결과

### 1) 사전조사 — claude-hud 데이터 흐름

- claude-hud 는 stdin (Claude Code 가 매 ~300ms 마다 statusLine 명령어로
  넘기는 JSON) 으로 `model.display_name`, `rate_limits.five_hour.{used_percentage, resets_at}`,
  `rate_limits.seven_day.{...}` 를 받음. 어디에도 영구 저장 안 함
  (`~/.claude/plugins/claude-hud/transcript-cache/*.json` 은 sessionTokens
  만, `config-cache/*.json` 은 settings sentinel 만).
- 결론: claude-hud 의 캐시 파일을 읽어서는 잔여량 못 얻음. **stdin 을
  가로챌 wrapper 가 필수**.
- 운영자와 합의: `A` (statusLine wrapper) 로 진행 + 멀티 계정 install /
  uninstall + claude-hud 병행 (있으면 pipe / 없으면 standalone).

### 2) 데자인 (Pencil + HTML mockup)

- `Docs/design/M0011-token-remaining-widget.pen` — 5 surfaces:
  - Dark HUD bars (default, 모델별 그룹)
  - Light HUD bars (임계 색상 demo: green→amber→red)
  - Text-only mode (custom color + transparent bg)
  - Settings panel (Theme/Layout 선택, Models per account 동적 토글)
  - Install dialog (per-account state, claude-hud 자동 감지, backup preview)
- Pencil **`+50px Y offset` 버그** 발견 — flex 레이아웃의 child 가 모두
  y=±50 으로 잘못 위치. 새 .pen 에서 alignItems:"center" 부터 깨짐.
  workaround: `layout:"none"` 으로 절대 좌표 배치.
- `Project/Plugins/token-remaining/mockup.html` — 동일 디자인의 HTML
  버전. 운영자가 브라우저로 직접 미리보기 가능.

### 3) 콘텐츠 결정 (운영자 피드백 반영)

- **Context % 제거** — 세션 단위라 계정 단위 위젯에서 분리. 5h/Weekly 만 표시.
- **모델별 그룹화** — Opus 4.7 / Sonnet 4.6 가 Claude Code 에서 별도
  rate-limit 윈도우를 가질 가능성 인지. 위젯에서 모델별 row 분리.
- **DB 백킹** — snapshot file → 30s collector → SQLite 영속 (시계열 지원
  여지 + dedupe). "마지막 값이 중요" 운영자 요건 충족.

### 4) 데이터 레이어 (ZeroCommon)

- `TokenRemainingObservation` — Vendor / AccountKey / Model / FiveHourPercent
  / FiveHourResetsAtUtc / SevenDayPercent / SevenDayResetsAtUtc / ObservedAtUtc.
- 인덱스: `(AccountKey, Model, ObservedAtUtc DESC)` — latest-per-pair
  lookup 핫패스.
- 마이그레이션 `20260504100119_AddTokenRemainingObservations` EF 툴링으로
  정상 스캐폴드 + 적용.

### 5) Collector + Query (ZeroCommon/Telemetry)

- `TokenRemainingCollector` 싱글톤. 30s 주기, 재진입 가드 (`_ticking`).
  매 tick: `cc-hud-snapshots/*.json` 전부 읽고 (Vendor, AccountKey, Model)
  의 latest row 와 (5h%, 7d%, resetsAt) 비교 → 변경 시만 INSERT (dedupe).
- 첫 5초 stagger 후 30초 cadence.
- App.OnStartup 에서 `Start()`, OnExit 에서 `Stop()` (5초 grace) 와이어링.
- `TokenRemainingQueryService` — `GetLatestForAccount`, `GetObservedModels`,
  `GetAccountProfiles`, `GetSeries`, `GetCollectorState`. EF 10 preview 의
  GroupBy.Select + record positional-ctor 한계 회피 위해 anonymous SQL
  group → ToList → 클라이언트 record 매핑 (M0009 와 동일 패턴).

### 6) StatusLine wrapper (AgentZeroWpf/Services)

- `StatusLineWrapperInstaller` — 4단계 install:
  1. `~/.claude*/settings.json` 읽기 (System.Text.Json + 트레일링 콤마/
     주석 허용) + 백업 (`%LOCALAPPDATA%\AgentZeroLite\statusline-backup\
     <acct>\settings.json.<ts>.bak`)
  2. wrapper.js 디스크 작성 (`%LOCALAPPDATA%\AgentZeroLite\statusline\
     az-hud-wrapper.js`)
  3. `statusLine.command` 교체 (기존 명령어 있으면 `--pipe "..."` 인자에 보존)
  4. install state 파일 작성 (`%LOCALAPPDATA%\AgentZeroLite\statusline-state\
     <acct>.json`) — sha256 of original + installed commands.
- Uninstall: install state 의 sha256 와 현재 statusLine 비교. 일치 시
  silent 복원, 불일치 (운영자가 settings 수동 편집) 시 `NeedsConfirm` 플래그
  + 3-way diff 위젯이 보여줌.
- `DiscoverProfiles()` — `~/.claude*` glob 으로 설치된 모든 Claude
  프로필 자동 발견 + 각각의 wrapper 설치 상태 + claude-hud 자동 감지.

### 7) wrapper.js (Node, 자동 작성)

- C# raw-string 으로 임베드 (별도 파일 번들링 불필요).
- 매 tick:
  1. stdin 읽기 + JSON 파싱 → model + rate_limits 추출
  2. 스냅샷 atomic 작성 (temp → rename)
  3. `--pipe` 있으면 자식 프로세스 spawn (stdin tee, stdout inherit)
  4. 없으면 minimal 한 줄 statusline 출력 (사용자가 빈 statusline 안 봄)
- 진단 로그: `cc-hud-snapshots/_wrapper.log` (rolling ~64KB)

### 8) WebDev Bridge RPC + JS

- `WebDevBridge` 에 `tokens.remaining.{profiles, accounts, observedModels,
  latest, series, status, refresh, install, uninstall, reset}` 10 종 +
  `tokens.remaining.tick` push event 추가.
- `zero-bridge.js` 에 `window.zero.tokens.remaining.*` 헬퍼 매핑.

### 9) 위젯 (Project/Plugins/token-remaining/)

- `index.html` + `widget.js` + `widget.css` + `manifest.json` + `mockup.html`.
- 위젯 본체: 모델별 그룹 row, 색상 임계 표시, 풋터 (DB rows + collector tick age).
- Settings 오버레이: Theme tabs, Layout 선택, Text-only 색상/투명도, Sync
  interval, **Models per account 동적 토글** (DB 에서 관측된 모델만), Wrapper
  install/uninstall per profile, Reset DB.
- Install/Uninstall 다이얼로그 — 3-way diff 보여주며 확인 받음.
- localStorage 기반 prefs 영속 (theme, hidden models, custom colors 등).

### 11) 진행 결과 — 작동 / 미해결

**작동 확인:**
- ✅ 위젯 UI 정상 렌더 (모델별 row, Settings 오버레이)
- ✅ Install/Uninstall 안전 (sha256 + 3-way diff confirm)
- ✅ `_wrapper.log` 가 매 tick 마다 stdin 파싱 + 모델 추출 + snapshot 쓰기
- ✅ DB 에 observation row 누적, 위젯이 latest 조회

**미해결 (다음 iteration):**
- ⚠ `claude-hud` chain pipe-through 가 `pipe exit=1` 로 종료. v2 wrapper
  에서 `cp.spawn('bash', [-c, script])` 직접 호출로 전환했으나 여전히
  bash 프로세스가 코드 1 로 종료. 원인 후보:
  - Git Bash 의 PATH 가 Node 자식 프로세스에 누락
  - `${plugin_dir}` 가 빈 문자열로 expand 되어 exec 실패
  - bash 가 우리 Node 의 환경 변수만으로는 plugin 경로 못 찾음
- ⚠ Opus / Sonnet 의 5h%/Weekly% 가 동일 값으로 관측됨 — Claude Code
  stdin 의 rate_limits 가 모델 무관 계정 단위 quota 일 가능성. claude-hud
  복구 후 시간차 비교로 검증 필요.

## 비고

- **C# `@""` raw-string 의 `"` escape 한계** — 처음 wrapper.js 를 `@""` 안에
  임베드하면서 JS 의 `\"` 를 그대로 썼는데, `@""` 에서는 `""` 로 escape 해야
  함. C# 11 `"""..."""` raw-string literal 로 전환해 escape 부담 제거.
- **Pencil 자동저장 부재** — `open_document` 로 만든 새 .pen 파일이 디스크에
  저장 안 됨 (운영자가 Pencil GUI 에서 Ctrl+S 해야 함). 이번엔 mockup.html 이
  대안 산출물 역할.
- **WPF dark theme + WebView2** — 위젯 자체는 dark 만 사용. WPF 컨트롤
  영향 없음 (M0009 의 Pitfall 6 우회).
- **claude-hud 가 bash 의존** — 일반적으로 plugin statusLine 은 cross-shell
  호환을 위해 단순 `node ${PLUGIN_ROOT}/...` 형태인데, 사용자의 claude-hud
  설치는 `bash -c '...'` 로 동적 버전 검색을 함. 이런 복잡한 쉘 명령은 우리
  wrapper 의 spawn 경로에서 fragile. 이상적으로는 wrapper 가 \"bash 같은
  쉘은 spawn shell 로 위임, 단순 명령은 직접 spawn\" 처럼 휴리스틱 분기 필요.
