# M0030 수행결과 — MP3 태깅 체계·무대 규칙 재정의·느낌 카드

## 실행 요약

M0029 로컬 MP3 플레이리스트의 2차 개선. 브리프 5개 축 모두 구현.

## 결과

### 1) 태깅 4계열 확립 — 장르 / 가수 / 음악적느낌 / 악기

- **느낌(Moods)**: AST AudioSet 무드 라벨 7종(happy/funny/sad/tender/exciting/
  angry/scary "~music")을 재생 중 실시간 누적 → `Mp3Track.Moods` CSV 영속
  (`mp3.setMoods`, 악기와 같은 클램프·dirty 재시도 경로). 나우플레잉·목록에
  💫 칩으로 표시.
- 장르(Category)·가수(Artist)·악기(Instruments)는 기존 유지 — 4계열이 검색과
  카드 필터의 공통 어휘가 됨.

### 2) 리얼타임 재조정 (사전분류보다 우선 가능)

- 재생 중 명시적 성별 라벨(Female/Male singing ≥ ACTIVE)이 **2틱 지속**되면
  무대 성별을 재조정 — LLM/비전 사전분류와 달라도 리얼타임이 이김.
- 단 **'그룹' 판정만은 유지** (오디오로 솔로/그룹 구분 불가). 곡 전환 시 리셋.
- 우선순위 최종: 그룹판정(불변) > 리얼타임(2틱) > LLM > 커버비전 > 기본 남성.

### 3) 무대 규칙 재정의

- **솔로 고정**: 재생 중 솔로 가수는 절대 무대에서 내려가지 않음 —
  남성: 매 틱 최소 PRESENT(0.05) 점수로 upsert(노래 신호 ≥ ACTIVE(0.12)일 때만
  노래 애니, 아니면 기본 idle 애니 — 점수 기반 상태 전환을 그대로 활용).
  여성: `upsertIdolGroup`에 pin — present 강제, 노래 애니는 기존 playing 플래그.
- **걸그룹 게이트**: `그룹 판정 && 신나는 음악` 동시 충족 시에만 group 모드 —
  신나는 음악 = 업비트 장르(K-Pop/EDM/힙합/록) or 리얼타임 exciting/댄스 라벨
  (10s 히스테리시스). 그룹 판정 곡의 잔잔 구간은 솔로 무대로 자연 전환,
  신나는 구간에 걸그룹 재등장. 신나도 솔로 판정이면 1인 모드(브리프 명시).
- 악기 등장 연출은 기존 그대로.

### 4) 검색 강화

- 통합 검색: 검색어가 가수/제목/느낌/악기(한·영) 어디든 매치. '단독' 모드는
  악기 검색어에만 적용(기존 계약 유지).
- **장르 → 가수 칩**: 1차 필터(장르 탭) 선택 시 하위 가수 칩(곡수 표시,
  상위 12) 원클릭 필터. 장르 변경 시 리셋.

### 5) 느낌 카드 자동추천 (🃏 카드 뷰)

- `Mp3MoodCard` 엔티티(Title/Description/FiltersJson) + 마이그레이션 — 카드는
  **저장**되어 재설치에도 유지.
- "➕ 카드 생성하기": 호스트가 DB에서 태그 인벤토리(장르·상위30 가수·느낌·악기)
  를 채굴해 LLM에 6줄 고정 형식으로 요청 → 응답의 모든 필터 값을 인벤토리에
  클램프(실존 값만) → 영속. 기존 카드 제목을 전달해 테마 중복 방지.
- 카드 = 제목 + 가수·느낌 요약 설명 + 필터 칩(장르/가수/느낌/악기) + 매치 곡수.
  클릭 → 풀에서 랜덤 재생, 곡 종료 시 같은 풀에서 자동 이어재생(자동추천 모드).
  목록에서 직접 곡 선택 시 카드 모드 해제. 🗑 삭제 지원.

## 운영자 스모크 가이드

1. 새 exe 재시작(마이그레이션 자동) + zip(v0.20.0) 재설치
2. 곡 재생 → 나우플레잉에 💫 느낌 칩이 쌓이는지 (신나는 곡이면 '신남')
3. 솔로 곡에서 가수가 간주 중에도 무대에 남아 있는지(idle 애니), 노래 구간에 노래 애니
4. 그룹 판정 곡(👥)에서 신나는 구간=걸그룹, 잔잔 구간=솔로 전환
5. 장르 탭 클릭 → 가수 칩 등장 → 원클릭 필터. 검색창에 가수명/느낌("신남") 입력
6. 🃏 카드 → ➕ 생성(LLM 필요) → 카드 클릭 → 랜덤 재생 + 자동 이어재생

> 커밋/푸시는 운영자 "잘 작동함" 확인 후 진행.

## 후속 수정 #1 — 3단 레이아웃 + 목록 전체/랜덤 재생 + 스캔 성능

**운영자 보고 (원문)**: "mp3 재생 레이아웃 2단(재생기,목록) 3단으로 해줄래 재생기,목록,
검색필터, 현재는 목록이 잘안보여서 3단으로 그리고 목록에는 이 목록 전체 순차재성 랜덤재생
기능도 ..검색기능 필터는 접혔다가 열릴수 있도록 ... 추가로 스캔중에 UI가 느린것같은데
스캔중에도 UI간섭을 덜해서 UI반응속도 우선 개선 ...스캔처리는 업데이트는 UI간섭없이 다만
스캔중 업데이트에대한 플레이항목처리가 항상 실시간일필요없음 스캔중에는 1분단위 리스트
업데이트되어도됨"

**진단 (스캔 중 UI 랙)**: ① 호스트가 파일당 1건씩 `mp3.track` 이벤트 발행 — 빠른 태그
스캔(수십/s)에서 WPF 디스패처 marshal + WebView2 postMessage 폭주. ② 플러그인이 이벤트마다
400ms 스로틀로 **전체 목록 innerHTML 재구성 + 아이템마다 리스너 재부착** — 대용량 목록에서
프레임 드랍.

**수정** (플러그인 v0.21.0):
- **3단 레이아웃** — `[미니 재생기 280px | 목록(남는 폭) | 검색·필터 패널 232px]`.
  검색창/장르 탭(`#mp3CatTabs` 신설 — 유튜브 탭과 분리)/가수 칩이 3단 패널로 이동,
  🔍 버튼으로 접힘(40px 레일)/펼침. 카드 뷰에서는 패널 숨김.
- **전체/랜덤 재생** — 목록 헤더 ▶ 전체(현재 필터 목록 순차재생 시작), 🔀 토글
  (켜면 곡 종료 시 필터 풀에서 랜덤 이어재생; ▶ 전체 시작곡도 랜덤).
- **스캔 성능**:
  - 호스트: 트랙 upsert 이벤트를 **배치**(2s 또는 100건, `mp3.tracks`)로 전환 —
    `Mp3TrackUpserted`(파일당 1건) 제거. 분류 패스 업데이트도 동일 배치.
  - 플러그인: 스캔 중 목록 렌더 **15s 스로틀**(운영자 허용 1분 이내; 진행 스트립은
    250ms 그대로), O(1) 인덱스맵 업서트(`mp3TrackIdx`), per-item 리스너 전부 제거 →
    `plList` **위임 리스너 1개**(목록/삭제/카드/카드생성 통합). 스캔 완료 시
    `mp3.list` 재로드로 정렬 복원 + 즉시 렌더.

**빌드 재검증**: 호스트 빌드 ✅ (0/0), 헤드리스 298 pass ✅, `node --check` ✅,
로컬 설치 zip(v0.21.0) 재생성 ✅.

## 후속 수정 #2 — 갱신 주기 스코프 축소 (최초 로드 지연 수정)

**운영자 보고 (원문)**: "최초 로드할때는 목록을 읽어와야하는데 1분갱신때문에 1분뒤
읽어오는것같음..1분갱신모드는 항상 작동할 필요없이 스캔중일때만 활성..그 이외에는
갱신내용이 없을것이기에 할필요없는듯해"

**진단**: 렌더 스로틀이 '스캔 여부'만 보고 일괄 적용 — 스캔 중이면 최초 목록 표시와
사용자 조작(검색 입력 등)까지 15s 타이머 뒤로 밀렸고, 이미 예약된 긴 타이머를 더 이른
요청이 앞당기지도 못했다.

**수정** (플러그인 v0.21.1, 호스트 무변경):
- 느린 주기(15s)는 **스캔 중 + 배치 데이터 유입에만** 적용. 스캔 아닐 때 400ms,
  사용자 조작(검색/필터)은 스캔 중에도 150ms.
- 더 이른 렌더 요청이 오면 대기 중인 긴 타이머를 **앞당겨 재예약**.
- **빈 목록에 첫 배치 도착 시 즉시 렌더** — 최초 로드가 스캔 스로틀에 걸리던
  직접 원인 수정.

**빌드 재검증**: `node --check` ✅, zip(v0.21.1) 재생성 ✅.

## 후속 수정 #3 — "재설치 시 스캔 데이터 소실" 보고 (착시) → 대용량 초기 로드 개선

그동안 빈 목록 문구("폴더 선택 후 스캔")가 보여 소실처럼 인지된 착시.

- **렌더 캡 300곡 + '▼ n곡 더 보기'** — 표시만 제한, 재생 순환·검색·카운트는
  전체 목록 기준. 필터/검색 변경 시 캡 리셋.

## 후속 수정 #5 — 카드 재생 확실화 (v0.24.1)

**운영자 보고 (원문)**: "방금 음악 카드 추천으로 생성성공함.. 하지만 해당 카드를 재생할
방법이 없음 클릭하면 해당 리스트가 가진 항목을 자동재생되게 해죠"

**진단**: 카드 클릭 위임 체인은 정상이나, ① 카드에 ▶ 버튼이 없어 재생 수단이 보이지
않았고, ② 생성 직후 라이브러리는 느낌/악기 태그가 아직 얕아 카드의 엄격 필터와 일치하는
곡이 0곡이면 경고만 뜨고 재생이 안 됨 ('0곡' 표기).

**수정** (v0.24.1, 호스트 무변경):
- 카드 헤더에 **▶ 재생 버튼** 명시 (본문 클릭도 동일 동작).
- **필터 점진 완화 폴백** — 엄격 일치 0곡 → 악기 → 느낌 → 가수 순 완화(장르 최후
  유지) → 전체 랜덤. 카드 클릭은 반드시 재생되며 완화 시 '≈ 유사 재생' 배지 표시.
  자동 이어재생(ended)도 완화 풀 사용, 카드의 '0곡'은 '≈ 유사 재생'으로 대체.
