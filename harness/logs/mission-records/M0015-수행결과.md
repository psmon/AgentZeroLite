### 진단

operator 의 핵심 통찰: **"명령보다 템포가 더 빠름"** — 강의 음성은 CLI
명령보다 빠른 템포로 흘러간다. 이전 라운드 (#1, #2) 의 Loose 튜닝
(80-frame / 4초 hangover) 은 "강의자 휴지를 단일 발화 안에 흡수" 라는
잘못된 가정으로 잡혔다. 실제 강의의 문장간 휴지(0.5–1.5초) 는 4초
hangover 를 절대 못 떨어트리고, 모든 문장이 mega-blob 으로 합쳐진다.
duration cap (25s) 이 backstop 으로 일하지만 그건 안전망일 뿐 sentence
boundary 가 아니다.

operator 가 원하는 것: **sentence-pace** segmentation. 문장당 한 줄로
note 에 저장하기 위해, 문장간 휴지가 hangover 를 떨어트려야 한다.

### 설계

| 요소 | 이전 (paragraph-pace) | 신규 (sentence-pace) | 근거 |
|---|---|---|---|
| Loose hangover | 80 frames (4.0s) | **24 frames (1.2s)** | intra-sentence micro-pause(~0.3s) > skip, sentence boundary(~0.8–1.5s) < trip |
| Loose preRoll | 1.5s | **0.5s** | 라인당 UI latency 단축 — operator 가 한 문장씩 이어 적힘을 즉각 확인 |
| Strict | 변경 없음 | 변경 없음 | 개인 클로즈마이크는 의도된 명령 페이스, 더 긴 hangover 가 자연스러움 |

새 invariant: **Loose hangover < Strict hangover** (이전엔 반대).
이전 라운드의 invariant ("Loose extends preroll/hangover") 는 잘못된
설계였음. 본 라운드에서 invariant 자체를 뒤집고 테스트도 갱신.
