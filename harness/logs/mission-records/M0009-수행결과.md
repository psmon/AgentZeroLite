# 수행 요약

토큰 모니터링 플러그인 (M0009) 풀 구현 완료. Claude Code 와 Codex CLI 가
디스크에 남기는 JSONL transcript 를 호스트 측 collector 가 10분 주기로
incremental 흡수하여 AgentZero SQLite DB 에 누적하고, WebDev WASM 플러그인
이 그 데이터를 대시보드로 노출한다. 플러그인은 수집/제어에 관여하지 않고
read-only.

## 결과

### 1) 텔레메트리 소스 사전조사

- **Claude Code CLI** — `~/.claude*/projects/<projSlug>/<sessionId>.jsonl`.
  `type=assistant` 라인의 `message.usage` 에 `input_tokens / output_tokens /
  cache_creation_input_tokens / cache_read_input_tokens` + `model` +
  `requestId` (dedupe key). `.credentials.json` 에 orgUuid (default profile만)
  + subscriptionType.
- **Codex CLI** — `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl`.
  `event_msg.payload.type == "token_count"` 의 `info.last_token_usage.*`
  + `rate_limits.plan_type`. 헤더(line 1) `session_meta.payload` 가 sessionId/
  cwd/cli_version/git 제공.
- **append-only** 보장 — 두 CLI 모두 파일을 truncate/rewrite 하지 않음.
- **멀티 프로필 발견** — operator 의 PowerShell `claude` / `claude-pel3` /
  `claude-qa` / `claude-gou` 매크로가 `CLAUDE_CONFIG_DIR` 로 home dir 분리
  사용 중 (총 4개 profile, ~410 JSONL). 단일 `~/.claude` 만 보면 절반 누락.

### 2) 데이터 레이어 (ZeroCommon)

- `TokenUsageRecord` — 한 LLM turn 한 row. vendor / accountKey / sessionId /
  cwd / gitBranch / model / recordedAt / 토큰 5종 / requestId / sourceFile / line.
- `TokenSourceCheckpoint` — 파일별 byte-offset 북마크. **`AccountKey` 컬럼이
  처음 ingest 시점에 pin** 되어 이후 `claude logout`/login 으로 active
  account 가 바뀌어도 이 파일의 row 들은 원래 계정으로 유지.
- `TokenAccountAlias` — `(Vendor, AccountKey) → Alias` 친화 별칭 테이블. UNIQUE.
- 마이그레이션 2개 (`AddTokenUsageRecords`, `AddTokenAccountAliasAndCheckpointAccount`)
  EF tooling 으로 정상 스캐폴드.

### 3) Collector (ZeroCommon/Telemetry)

- `TokenUsageCollector` 싱글톤. `Start()` → 5초 stagger 후 10분 cadence
  `System.Threading.Timer`. 재진입 가드 (`_ticking` 플래그) 로 직전 tick
  미완료 시 다음 fire skip.
- **다중 profile 자동 탐색** — `~/.claude*/projects/` glob → AccountKey =
  leading dot 제거한 dir 이름 (`claude`, `claude-pel3`, `claude-qa`,
  `claude-gou`).
- 파일별 `(ByteOffset, LineCount)` checkpoint 로 incremental — 다음 tick은
  새 라인만 읽음. `FileShare.ReadWrite|Delete` 로 producing CLI 와 공존 안전.
- Claude row 필터: `type=assistant && message.usage`. 0-row keepalive 라인 skip.
  Codex row 필터: `event_msg && payload.type=token_count`. `last_token_usage`
  우선, 없으면 `total_token_usage`.
- `ResetData()` — `ExecuteDelete()` 로 Records + Checkpoints wipe → 다음 tick
  부터 offset 0 부터 재흡수. 대시보드 Reset 버튼이 호출.
- App.OnStartup 에서 `Start()`, OnExit 에서 `Stop()` (5초 grace) 와이어링.

### 4) Query 서비스 (ZeroCommon/Telemetry)

`TokenUsageQueryService` — Totals / ByVendor / ByAccount / ByProject /
TimeSeries / Sessions / Recent / CollectorState / Profiles / Aliases.
EF 10 preview 가 record positional-ctor projection 을 GroupBy.Select 안에서
번역 못 하는 한계 때문에 **익명 타입으로 SQL group + ToList → 클라이언트에서
record 매핑** 패턴으로 통일. `ByProject` 는 `Path.GetFileName` 이 SQL 비번역
이라 materialize-then-group 클라이언트 처리.

### 5) WebDev Bridge (AgentZeroWpf)

`WebDevBridge` 에 `tokens.*` RPC 11종 (`summary / byVendor / byAccount /
byProject / timeseries / sessions / recent / refresh / status / profiles /
aliases / aliases.set / aliases.remove / reset`) + `tokens.tick` push event.
`zero-bridge.js` 에 `window.zero.tokens.*` 헬퍼 매핑.

### 6) Token Monitor 플러그인 (Project/Plugins/token-monitor/)

- 카드 4개 (총 토큰 / Anthropic 합 / OpenAI 합 / Collector 상태)
- 라인 차트 — vanilla canvas 2-series, 차트 라이브러리 의존성 없음, dark theme
- 표 3개: By vendor·account (alias inline rename) / **By project** (CWD leaf
  폴더명 그룹) / Recent sessions
- range selector (1h/6h/24h/7d/30d/all) + Refresh / Reset & Re-scan
- `tokens.tick` 자동 구독 → 매 cycle 자동 갱신
- IndexedDB 미사용 (read-only DB-source), 호스트 SQLite 가 단일 source

### 7) Install Plugin 다이얼로그 개선

- `OfficialPluginCatalog` 신규 — github.com/psmon/AgentZeroLite 의
  `Project/Plugins/` 를 GitHub Trees API 로 런타임 탐색 → 각 폴더의
  `manifest.json` 을 raw URL 로 fetch 해 `OfficialPluginEntry(Id, Name,
  Version, Description, Icon, GitFolderUrl)` 리스트. 하드코딩 제로.
- 다이얼로그 리뉴얼: 상단 OFFICIAL 콤보 + 메타 패널 (icon/name/version/id/
  description/url) + Reload(↻), 하단 CUSTOM (zip + 수동 Git URL 입력).
- 다크 테마 ComboBox 풀 재템플릿 (DarkComboBoxToggleStyle / DarkComboBoxItem /
  DarkComboBox) — system color fallback 제거.

## 비고

- **수집 주기 1분 → 10분 변경** — operator 제안. 재시작 gap-fill + Refresh
  버튼이 즉시성 커버하므로 부하 1/10 감소.
- **WPF 다크테마 사고 재발 방지** — 사용자가 "여러 번 실수" 명시적으로 언급 →
  Pitfall 6 + memory + code-coach pre-commit 체크리스트 update.
- **EF 10 preview LINQ 한계** — record positional-ctor projection 을
  GroupBy.Select 에서 번역 못함. 모든 group-by 쿼리를 익명타입 우회로 통일.
  EF 10 GA 후 다시 평가.
- **Claude orgUuid bug** — 첫 구현이 `claudeAiOauth.organizationUuid` (nested)
  로 찾아서 fallback 으로 "max" 가 모든 row 에 들어갔음. 실제로는 top-level.
  multi-profile 발견 후 어차피 이 로직을 폐기 (AccountKey = profile dir 이름)
  하고 `ResolveClaudeAccount()` 함수 자체 제거.
- **In-flight tick race** — Refresh 버튼이 timer-driven tick 과 동시에 같은
  파일을 읽을 가능성 (rare). 일상 사용에서 미미해 fix 보류. 발현 시 `_ticking`
  가드를 manual path 도 공유하게 묶으면 됨.
