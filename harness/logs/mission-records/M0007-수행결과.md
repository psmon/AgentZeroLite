# 수행 요약

M0007 의 본 mission scope 인 **WebDev 플러그인 인프라**를 마감했다. 음성노트
플러그인 본체는 **M0008** 로 분리해 inbox 에 등록 — 이번 미션은 그 위에서
M0008 이 그대로 빌드될 수 있는 인프라까지만.

세 줄로:
1. **git URL 설치 채널** — GitHub 트리 URL 만 붙이면 `manifest.json` fetch →
   검증 → 폴더 전체 파일 다운로드 → 마운트. CLI git 의존 없이 raw HTTP +
   GitHub Trees API.
2. **+ Install Plugin UI 확장** — 모드 선택 다이얼로그로 ZIP / Git URL 둘 중
   하나를 명시 선택.
3. **음성노트 native API** — `window.zero.note.*` (start/stop/pause/resume/
   set-sensitivity) + `window.zero.summarize`. VAD 가 발화 segment 닫을 때마다
   STT 자동 transcribe → JS 로 `note.transcript` event push. Max-token 회피는
   문자 길이 기반 1/2 분할 재귀 요약.

추가로 `Project/Plugins/` 디렉토리 + 카탈로그/설치 가이드 README 신설. 컴파일
포함되지 않는 외부 플러그인 프로젝트 보관소.

### B. 확장된 기존 파일

- **`WebDevPluginInstaller.cs`** —
  - 정적 `HttpClient` (User-Agent 포함, 30s timeout) + 가드 상수
    (`MaxFileBytes=25MB`, `MaxFiles=200`).
  - `InstallFromGitUrlAsync(url)` 신규:
    1. URL 파싱 (browse `tree/{branch}/{path}` 또는 raw `manifest.json` 패턴)
    2. raw `manifest.json` fetch → `PluginManifest.Parse` (ZIP path 와 동일
       검증).
    3. `git/refs/heads/{branch}` 로 SHA 해석 → `git/trees/{sha}?recursive=1`
       으로 전체 트리 → prefix 로 폴더 필터.
    4. 파일별 raw URL 로 다운로드, 사이즈 가드 + zip-slip 가드 (resolved
       path 가 staging dir 안에 있는지 확인).
    5. entry 파일 sanity check → atomic move into
       `%LOCALAPPDATA%/AgentZeroLite/Wasm/plugins/{id}/`.
    6. 실패 시 staging 정리 (`finally`).
  - 정규식 두 개 (browse / raw) 로 GitHub URL 양 형식 흡수.
- **`WebDevHost.cs`** — 음성노트 surface 추가:
  - `StartNoteCaptureAsync(int? sensitivityPercent)` — `VoiceCaptureService`
    + `BuildStt(VoiceSettingsStore.Load())`. STT preload (`EnsureReadyAsync`)
    실패 시 `NoteError` 이벤트로 사용자 안내 후 false 반환.
  - `OnNoteUtteranceEnded` 핸들러 — `ConsumePcmBuffer` → 별도 Task 에서
    `TranscribeAsync` (capture thread block 회피) → `NoteTranscript` 이벤트.
  - `Pause/Resume` = `Muted` toggle, `SetNoteSensitivity` =
    `VadThreshold = SensitivityToThreshold(percent)`.
  - `SummarizeAsync(text, maxChars=6000)` + `SummarizeRecursiveAsync` —
    문자 길이 초과 시 `FindSentenceBoundary` 로 가장 가까운 sentence
    terminator 에서 분할 → 각 절반 재귀 요약 → 최종 한 번 더 consolidate.
    재귀 깊이 ceiling 6 (무한 폭주 가드).
  - One-shot session (`await using`) 으로 cached `_chatSession` 오염 방지.
  - `SummarizeResult` record 신설.
  - 4 events: `NoteTranscript` / `NoteUtteranceStarted` / `NoteUtteranceEnded`
    / `NoteError`.
- **`WebDevBridge.cs`** —
  - 생성자에서 `host as WebDevHost` 로 캐스트 → 4 events subscribe (Detach 시
    unsubscribe). `WebDevHost` 가 아닌 다른 `IZeroBrowser` 구현 (테스트 등)
    에선 note.* op 가 명시 에러 ("voice-note surface unavailable").
  - 새 op 7 개: `note.start`, `note.stop`, `note.pause`, `note.resume`,
    `note.set-sensitivity`, `note.status`, `summarize`.
  - 새 event 4 채널: `note.transcript`, `note.utterance-start`,
    `note.utterance-end`, `note.error`.
- **`WebDevPagePanel.xaml.cs`** — `OnInstallPluginClick` async 화 + 두 모드
  분기. Git 모드 진행 중엔 기존 `ShowLoading(detail)` 재사용해 사용자에게
  진행 시각 피드백.
- **`Wasm/common/zero-bridge.js`** — `window.zero.note.*` + `window.zero.summarize`
  wrapper 추가. JS 입장에선 callback 등록 (`onTranscript` 등) 으로 깔끔하게
  사용 가능.
- **`Wasm/README.md`** — JS API 섹션을 core / voice-note 두 블록으로 재정리.

**알려진 한계 / 후속 정리 후보**:
- VoiceCaptureService 는 mic singleton — Settings/Voice 탭이 capture 중인
  상태에서 음성노트가 start 하면 마이크 conflict 가능. 현재는 명시 핸들링
  없음. M0008 스타트 직전에 conflict detector 추가 검토.
- Summarize 가 토큰 단위 정확 측정 없음 (LlmGateway 에 tokenizer 없음).
  문자 길이 (6000) 가 proxy. 한국어/영어 1자 ≈ 1~2 토큰 가정. 8k window
  안전 범위. 정확한 tokenizer 통합은 별도 후속.
- git URL 인스톨러는 public GitHub 만 지원. private repo / GitLab / 자체
  호스팅은 후속 (PAT 인증 + 다른 API 형식).
