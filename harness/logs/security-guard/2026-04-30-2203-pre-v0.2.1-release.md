## Scope Item 1: Injection Surface ‚Äî PASS

### `WriteAndEnter` (cbee04e) ‚Äî equivalent to `WriteAndSubmit`

**File:** `Project/AgentZeroWpf/Services/ConPtyTerminalSession.cs:160‚Äì169`

```csharp
public void WriteAndEnter(string text)
{
    Write(text.AsSpan());
    _ = Task.Delay(200).ContinueWith(_ => SendControl(TerminalControl.Enter),
        TaskScheduler.Default);
}
```

**Comparison vs existing `WriteAndSubmit`:**
- Same `Write(text.AsSpan())` ‚Üí `_pty.WriteToTerm(text)` path.
- Same set of callers (AgentBot ChatMode user-input sites, slash-command
  exec, /skills sync, slash menu) ‚Äî these were already calling
  `WriteAndSubmit` with identical privileges.
- Difference is purely a timing/dispatch tweak: 200ms gap (was 50ms),
  Enter routed through `SendControl(TerminalControl.Enter)` instead of
  inline `WriteToTerm("\r")`. Both ultimately reach the same
  `_pty.WriteToTerm` write path.

**No new privileged command added. No auth bypass. No escalation.** The
existing v0.2.0 finding (Medium ‚Äî STT transcript shell-escape) applies
identically to both methods because both go to `Write` ‚Üí `_pty`. Status
unchanged.

### Echo-check diagnostics (cbee04e, f721ebd) ‚Äî read-only

`ScheduleEchoCheck` and `NoteInputAttempt` only **observe**
`_pty.ConsoleOutputLog.Length` and emit `AppLogger.Log` lines. No
additional bytes are written to the PTY. Health-state transitions
(`Alive`/`Stale`/`Dead`) are signalling only ‚Äî no auto-write recovery.

### Wedge-recovery banner + Restart Terminal (0bc0b35) ‚Äî privilege-neutral

`RestartWedgedTerminal` disposes the existing `ITerminalSession`, calls
`terminal.RestartTerm()` (which spawns a fresh PTY child via the same
existing path used at first-start), and rebinds. This is the same
privilege envelope the user already has ‚Äî they could close and reopen
the tab manually for the same effect. **No finding.**

### Click-to-target (baca2fd) ‚Äî pure UI routing

`MarkTabAsBotTarget` updates `_activeConsoleTab` and notifies the actor
system via `SetActiveTerminal`. No code paths execute commands; the
worst-case is "AgentBot sends its next message to a different tab than
the user intended" ‚Äî a UX bug, not a security one. The reentrance flag
(`_targetingFromTerminalFocus`) prevents dispatcher chains. **No finding.**
