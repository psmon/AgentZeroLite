# Security-Guard Pre-Release Audit — v0.2.0

## Executive Summary

AgentZero Lite v0.2.0 introduces a substantial voice subsystem (Whisper.net
+ Vulkan GPU + NAudio mic capture + OpenAI/Webnori cloud STT/TTS +
Akka.Streams pipeline) and interactive PixiJS demos on the Home page.
Audit confirms the injection surface remains defensible, actor name
sanitization is in place, IPC is validated, and dependencies are signed
from official sources. **One Medium finding** identified (voice transcript
shell-escaping) and **one Low finding** (GGML download hash verification
not visible in code). Neither blocks the release per the gate rule
(only Critical/High block).

---

## Scope Item 1: Injection Surface

### MEDIUM — STT Transcript → Terminal Without Escaping
**CWE:** CWE-78 (OS Command Injection), CWE-94 (Improper Control of Generation)

**File:** `Project/AgentZeroWpf/UI/APP/AgentBotWindow.Voice.cs:320–322`

```csharp
txtInput.Text = transcript;
txtInput.CaretIndex = transcript.Length;
SendCurrentInput();  // → session.WriteAndSubmit(text)
```

**Root cause:** STT transcript is assigned directly to the chat input and
auto-sent without an escaping checkpoint. A poisoned STT backend (MITM
on cloud STT, corrupted local GGML, hostile mic feed) could embed shell
metacharacters that execute in the active terminal — bypassing the AIMODE
guardrail.

**Risk:** Low likelihood (requires backend compromise or mic poisoning),
but high impact (RCE via voice channel).

**Remediation (post-release):**
1. Wrap transcript in PowerShell single-quote escape, or
2. Add an opt-in voice-confirmation dialog that displays the transcript
   before send.

**Why not blocking:** The path requires upstream compromise of OpenAI /
Webnori / a corrupted GGML model file. The CHT mode is functionally
identical to typing in the chat box (which the user implicitly trusts).
Issue logged for v0.2.1.

### INFO — Auto-Handshake Peer-Name in Terminal Title
**File:** `Project/AgentZeroWpf/Services/WorkspaceTerminalToolHost.cs:110, 164–187`

`peerName` (terminal tab title) is embedded in the auto-prepended handshake
header. Verified the value is only used in:
- Log lines (`AppLogger.Log`)
- Akka actor messages (`MarkHandshakeSent`, `MarkConversationActive`)
- The handshake header text body (sent **to** the peer terminal, not
  executed locally)
- `--from` parameter in `bot-chat` template, which is JSON-escaped at
  `CliHandler.cs:717` via `EscapeJson()`

No shell-metacharacter injection path exists. Tab titles in WPF TabItem
are sanitized indirectly by the UI control. **No action required.**

### INFO — Voice AIMODE Path Already Guarded
AIMODE-driven `SendToTerminalAsync` originates from Gemma/Nemotron LLM
output passed through GBNF grammar constraints. The voice path lacks an
equivalent layer (see Medium above), but AIMODE itself is unaffected.

## Scope Item 2: Actor Name Sanitization — PASS

**Files:** `Project/ZeroCommon/Actors/ActorNameSanitizer.cs`,
`StageActor.cs`, `WorkspaceActor.cs`, `VoiceStreamActor.cs`

- `ActorNameSanitizer.Safe()` converts non-ASCII / disallowed Akka chars
  to `_`, appends `-{hashcode}` on collision (lines 14–35).
- `StageActor` calls `Safe()` on workspace names before path construction.
- `WorkspaceActor` calls `Safe()` on terminal IDs.
- `VoiceStreamActor` uses a hardcoded path (`/user/stage/voice`); no user
  input reaches the actor name.

**No new sanitization gaps introduced in v0.2.0.**

## Scope Item 3: CLI ↔ GUI IPC — PASS

**Files:** `CliHandler.cs:85–142, 679–742`, `MainWindow.xaml.cs:410–420, 726–771`

- `WM_COPYDATA` marker `0x414C` ("AL") validated on receive.
- JSON payload parsed via `JsonDocument.Parse()` with exception handling.
- `EscapeJson()` applied to all user-controlled strings before
  serialization.
- `bot-chat --from <peerName>` parameter is JSON-escaped in the same
  path. Reverse-channel input from peer terminals goes through the same
  validation as direct CLI invocations.

### LOW — GGML Model Download Hash Verification Not Visible
**File:** `WhisperLocalStt.cs:94–97`

```csharp
await using var modelStream = await WhisperGgmlDownloader.Default
    .GetGgmlModelAsync(ggmlType, cancellationToken: ct);
```

`WhisperGgmlDownloader.Default` is a Whisper.net library internal —
hash verification (if any) is not visible in our code. The local
truncation guard (`IsModelDownloaded` checks file length against
`minBytes`) protects against partial downloads but not malicious
substitution.

**Remediation (post-release):** Open an upstream issue with Whisper.net
maintainers to confirm SHA256 verification, or pin to known-good model
hashes locally.
