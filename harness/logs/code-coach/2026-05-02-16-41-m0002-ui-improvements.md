## Naming and small-scale design

- `BuildRedockStrip` / `RedockOneTab` / `RefreshAllRedockStripVisibility` /
  `CloseEmptyFloatingWindows` / `CountLayoutDocuments` — all communicate
  intent clearly.
- DETACH-only on main window's CONSOLE strip / REDOCK-only on floating window
  is asymmetric on purpose: matches user mental model
  ("detach this from here" vs "come back from here").
- 3-tier close strategy in `CloseEmptyFloatingWindows` is overkill *if* the
  Layout.RemoveChild path always works, but the diagnostic value (knowing
  which strategy succeeded) justifies it. AvalonDock's behavior may shift on
  NuGet upgrade; the ladder + log will surface that.
