## Review by lens

**.NET modern** — Same `[SkippableFact]` + `Skip.IfNot(File.Exists(...))`
pattern as T0/T1G/T2G. List/loop in T4G is straightforward. Nothing exotic. ✓

**LLM / native** — T4G uses `MaxIterations = 1` per trial to keep stability
runs fast (single inference each). T3G uses `MaxIterations = 6` for the
multi-step chain. Both reuse the loaded LLM across trials/calls (model load
is amortized once per test method). ✓

**Test-design notes** —
- T3G uses a *soft assertion* on tab index (logs whether routing was
  correct, fails only if NO send_to_terminal happened) — appropriate for
  single-trial semantic at 4B-class scale.
- T4G uses a *statistical assertion* (≥ 4/5) — the right shape for
  non-deterministic LLM output. GBNF guarantees structural validity 100%
  but tool selection is the model's free choice.
- Both tests log per-call detail via `_output.WriteLine` so failures are
  diagnosable from console history.
