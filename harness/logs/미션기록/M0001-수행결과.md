## 비고

### 디렉터리 이름 `미션기록` 에 대해
하네스 전체 정책은 "모든 artifact 영어"지만, 이 디렉터리는 의도적으로 한국어로
두었다. operator의 기록 캐비닛이고, 결과 파일들은 전부 operator 언어를 따르므로
컨테이너 이름도 같은 언어가 자연스럽다. tooling 호환성이 필요해지는 시점에 영어
별칭(`logs/mission-records/`)을 심볼릭 링크나 alias로 더할 수 있으나 지금은
단순함을 우선했다.

### bootstrap 자기참조에 대해
M0001은 missions 시스템 자체를 만드는 미션 — 기록 인프라가 아직 없는 상태에서
기록 인프라를 만드는 self-bootstrap 케이스다. 시스템이 만들어지는 *과정*이
시스템의 *첫 번째 use case* 가 되는 형태로, 향후 트러블슈팅 시 이 로그를
canonical example로 참고할 수 있다.

### operator raw draft 발견 및 처리
실행 도중 operator가 미리 staging해 둔 `harness/missions/M001-하네스-미션구조생성.md`
(3자리 ID + 한국어 파일명, frontmatter 없음)를 발견했다. 내용은 이번 요청과 동일.
정식 형식의 `M0001-add-missions-system.md`(4자리 + 영어 slug + frontmatter)가
canonical version이고, raw draft는 archeological reference로 그대로 보존했다.
다음 미션부터는 README.md의 명명 규칙을 따른다. operator가 정리를 원하면
`M001-…md`는 삭제 가능 — 별도 정보 손실 없음.

### 다음 미션부터의 운용
사용자가 예고한 대로 다음부터는 M0002, M0003 … 으로 짧게 호출 예정.

   "다음 미션 번호" / "M{NNNN} 미션 만들어" 식으로 스캐폴드 요청).
2. operator: "M{NNNN} 수행해"
