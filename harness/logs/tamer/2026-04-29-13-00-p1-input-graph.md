# P1 완료 — INPUT 그래프 (마이크 → STT → AgentBotActor)

## 실행 요약

- 액터-매개 음성 입력 파이프라인 도입. 기존 batch 경로(`OnVoiceUtteranceEnded → SendCurrentInput`) 는 그대로 유지하고, `VoiceSettings.UseStreamPipeline` 플래그로 새 경로를 옵트인.
- 새 경로 토폴로지: `Source.Queue<MicFrame>(64, DropHead)` → `KillSwitches.Single` → `VoiceSegmenterStage` (custom GraphStage) → `.Async()` → `SelectAsync(parallelism)` → `SttWorkerActor` (SmallestMailboxPool) → `Where(non-empty)` → `Sink.ActorRefWithAck(Self)`.
- `StatefulSelectMany`가 Akka.NET 1.5의 FlowOperations 확장에 미정의여서 (SourceOperations 만 보유) **custom GraphStage<FlowShape>** 로 직접 구현. inlet/outlet pull-push 핸들러로 backpressure 정확히 보존.

## 결정 / 트레이드오프

- **Custom GraphStage 채택 사유**: Akka.NET 1.5 `FlowOperations` 에 `StatefulSelectMany` 미존재. `Scan + SelectMany` 우회보다 GraphStage 가 더 명시적이고 단위 테스트 가능 (TestKit Probe 와 자연스럽게 결합).
- **`DropHead` 선택 사유**: 마이크는 실시간 디바이스, backpressure 적용 시 OS 레벨 buffer overflow 가능. 최신 프레임을 보존해 sttPool 회복 시점에 가장 가까운 음성 처리.
- **STT 풀 size = `VoiceSettings.StreamSttParallelism`**: 기본 1. 동시 발화가 거의 없는 1인 세션이라 충분. 향후 다중 사용자 / 회의 모드 시 늘림.
- **Shadow / dual-run 미채택**: 비용 (STT API 비용 + 로컬 GPU 부담) 부담으로 옵트인 토글로 단일 경로만 동작.
